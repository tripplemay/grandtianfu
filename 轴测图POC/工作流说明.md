# 户型图 → 轴测图 → 效果图 · 工作流

## 一句话
**改家具只在一个地方(`furniture-D户型.json` 或可视化编辑器),一条命令出全部图,提示词自动同步。**

## 文件角色
| 文件 | 作用 |
|---|---|
| `平面布置图-生成器.py`(根目录) | 几何源:出 `平面布置图-无家具.svg`(墙/窗/房名)。**只在改墙/窗时跑** |
| `furniture-D户型.json` | **家具方案(唯一数据源)** —— 每件 `{t,x,y,w,h,orient,...}` |
| `轴测引擎.py` | 可复用引擎(户型无关):几何+家具 → 2D平面 / 轴测照片 / 轴测空壳 |
| `build.py` | **一键构建**:出 2D平面 + 两张轴测 + PNG + 自动提示词,并打开预览 |
| `editor.html` + `serve.py` | **可视化拖拽编辑器**:拖家具→保存→自动重出图 |
| `prompt_gen.py` | 从家具表自动生成 4D 提示词(随家具同步) |
| `图生图-提示词.md` | 图生图四种方案说明(4A/4B/4C/4D) |
| `legacy/` | 旧脚本归档 |

## 三种常用操作

### A. 可视化编辑(最方便,双模:家具 + 几何/墙体)
```bash
cd 轴测图POC && python3 serve.py        # 浏览器自动打开编辑器
```
顶部切 **[家具]/[几何]** 两个 Tab:
- **家具模式**:拖动家具改位置 → 选中右侧改 类型/尺寸/朝向/标签 → 「💾 保存并重出图」。
- **几何模式(墙体/户型)**:拖房间矩形=移动、8 把手=缩放(Alt 关吸附);墙体由房间边**自动派生**(实时预览);「打通/分隔」并/拆空间(墙随之增删);「＋门(点墙)」在墙上插门、「＋自由墙」画两点;门窗为沿墙滑块(改 span/类型/开向/落地)。校验拦截(跨空间重叠、开洞悬空墙外等)→「💾 保存并重出图」。
> 几何=数据驱动:唯一手编层 `geometry-D户型.json` →`geometry.py derive()` 自动推墙(L轮廓/共享边去重/连通开放/开洞切墙/内外墙厚/虚线公共墙)→ 2D平面(含自动尺寸线+门弧)与轴测同源。

### B. 命令行调家具
直接改 `furniture-D户型.json` 一行,然后:
```bash
cd 轴测图POC && python3 build.py D        # 出全部图并打开
# 只改了家具没改墙:python3 build.py D --no-geom (更快)
```

### C. 出写实效果图
1. 跑 build(或编辑器保存)→ 得到 `D户型-照片底图.png` + `D户型-4D提示词.txt`(已自动同步家具)
2. 把 **照片底图** 拖进 Gemini / Nano Banana,贴 **4D提示词.txt** 的内容 → 出写实图
3. (要锁结构走 ControlNet 见 `图生图-提示词.md`)

## 输出
- `平面布置图.svg/.png`(根目录,2D 含家具平面)
- `D户型-照片底图.svg/.png`(轴测·写实底图)
- `D户型-空壳底图.svg/.png`(轴测·空壳)
- `D户型-4D提示词.txt`(自动)

## 新增户型
1. 准备该户型精准几何 SVG(或用生成器改)→ 得到 `xx-无家具.svg`
2. 复制 `furniture-D户型.json` → `furniture-X户型.json`,按新平面填家具
3. 在 `build.py` 的 `HOUSES` 加一行(指向新 SVG + 新 json)
4. `python3 build.py X` —— 引擎/编辑器/提示词全部复用,一行引擎都不用改

## 坐标约定
1 单位 = 10mm(例 w=300 即 3.0m);原点同平面图(轴测图内部已处理 translate(150,250) 偏移)。
