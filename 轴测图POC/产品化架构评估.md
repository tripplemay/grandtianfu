# 阅天府软装工具链 → 产品化:最终架构与整理方案

## 0. 一句话总判断

你手里是**一个产品级几何内核 + 一批已验证的交互/提示词资产**,被包在**一层单机原型外壳**里。演进的本质是"抽内核、换外壳",**不是重写**。但五视角的技术方案集体犯了三个乐观偏差,经对抗挑战修正后,最终方案的基调是:**先验证商业价值,再上平台;先把确定能做的(#2/#3/#5/#6)产品化,把决定生死的(#1/#7)关进时间盒试点;绝不把研究级不确定性放进 MVP 关键路径。**

## ① 技术栈诚实裁决表

| 层 | 现有实现 | 裁决 | 理由(已实读核实) |
|---|---|---|---|
| 几何内核 | `geometry.py`(552行 derive/validate) | **保留·升格为库** | 纯函数、无副作用、derive 实测~20ms、有黄金回归。是唯一真护城河。**但仅支持轴对齐矩形**(见对抗③/⑬/㉘,这是产品边界不是脚注)。 |
| 轴测/2D 渲染 | `轴测引擎.py`(651行) | **保留·切割+去硬编码** | 数据驱动 SVG 生成是资产;但删 legacy `parse_geometry`(正则解析 SVG)、外提 D户型特化常量(`at>=1390` 满窗、`WALL_H=1450`)、`render()` 改返回字符串。**注意:此层无黄金测试覆盖(见对抗㉖),改前必须先补渲染快照测试。** |
| 提示词生成 | `prompt_gen.py` | **逻辑保留·知识外置** | 风格/材质模板(`MAT`/`NAME_MAT`)外置为数据(风格库);类型词表并入统一家具目录。 |
| PNG 光栅 | `rsvg-convert` subprocess | **机制保留·服务端化** | **维持 rsvg + 容器内嵌 Noto Sans CJK**(中文房名硬约束)。resvg 是诱惑陷阱:引擎重度依赖 feGaussianBlur/glow/feMerge/pattern/gradient,resvg 滤镜支持不全会**静默丢辉光阴影**(对抗⑧/⑰)。resvg 只能在对黄金图做像素级 diff 验证通过后作为可选加速项。 |
| 服务层 | `serve.py`(stdlib http.server) | **替换** | `HOUSE="D"` 硬编码、subprocess 同步重出图、多用户写同一组文件必互相覆盖。但路由形状(`/derive`/`/save-geometry`)可 1:1 映射到 FastAPI。 |
| 编辑器 | `editor.html`(647行 vanilla JS) | **逻辑保留·框架化** | 双模拖拽/吸附/实时 derive 预览/跨space校验逻辑完整,是优秀的需求规格。重写为 React + 受控 SVG,**不上 Pyodide/WASM**(对抗⑨/⑱:derive 仅20ms,HTTP+debounce 足够,Pyodide 是为边际延迟换永久架构枷锁)。 |
| AI 调用 | 人工粘贴 Gemini + 网关 | **重建·按需求分流** | 人工不可规模化。**网关 img2img 经 MCP 工具表面不可达(`generate_image` 入参无图片字段)** —— 此为五视角间的事实冲突,裁定:**#4 走网关 chat(今天可用);#6/#7 直连 provider(fal/Replicate/Gemini 图像编辑)**,不依赖网关图片入口(对抗⑦/⑯)。 |
| 数据模型 | 两个 JSON 文件 | **演进·三处债先修** | 见 ④/④数据债。最大单点债是 furniture 的复合字符串外键(对抗㉔/㉕)。 |

## ② 目标架构(文字图)

**裁定单一目录约定**(终结五视角五套结构的分歧,对抗㉙):所有可部署单元放 `apps/*`,所有库放 `packages/*`,**Python 包目录一律下划线**(`floorplan_core`,连字符仅作 pyproject 发行名);引擎先合为**一个包**(geometry+render+prompt 同包内分模块),只有当渲染需独立 GPU 扩缩时才拆出 `services/render`。否定顶层散装 `engine/ai/api/web`。

```
后端(MVP = 单容器单体,不是微服务)
  FastAPI 单体  ──直接 import──> floorplan_core(纯 Python 引擎包)
    ├─ /derive /validate          [同步 ~20ms 纯函数,热路径]
    ├─ /save-geometry /furniture  [同步,写版本快照,不覆盖]
    ├─ /style-chat (#4)           [同步/SSE,走网关 chat + json_mode]
    └─ /renders /ai-renders       [入队 → 异步]
  任务队列  Redis + RQ(1 worker 起步,不要 Celery 全家桶)
    ├─ SVG→PNG 光栅 (rsvg + 内嵌CJK字体)   确定性,内容哈希缓存命中率高
    ├─ #6 img2img (直连 provider + ControlNet锁线稿)
    └─ #7 inpaint (直连 provider,Phase 2+)
  存储  起步 SQLite/文件(目录即数据模型) → 规模化 Postgres(JSONB)+ 对象存储(S3/R2)
  鉴权  起步可无 → 托管(Clerk/Supabase),不自建 RBAC

前端  React + 受控 SVG(SVG = render(derive(G)),声明式同构)
       家具层拖拽性能吃紧再下沉 react-konva。不上 three.js。

同步/异步边界(明确):
  同步(HTTP):derive、validate、保存版本、2D/轴测 SVG 生成、#4 风格对话
  异步(队列+WS/轮询):PNG@3200光栅、#1 raster→vector、#6 img2img、#7 inpaint
```

**一次"户型图→效果图"请求时序:**
```
上传位图 → API存原图 → [异步]#1 vision-LLM出草稿geometry → validate兜底
  → 用户在编辑器手工拉房间/单基准标定尺寸(#2,必经,非自动)
  → 保存(不可变新版本) → [异步]#3 轴测SVG→PNG
  → 描述风格(#4 网关chat→家具清单+风格,LLM不吐坐标)
  → 模板布局落坐标 → 编辑器微调(#5)
  → [异步]#6 prompt_gen+轴测线稿底图→直连provider img2img(ControlNet锁线)
  → 人工审核挑图 → 交付
  (#7 独立管线:空房照→SAM掩膜+depth→inpaint,与几何引擎无关,见③)
```

## ③ 7 需求逐项技术选型

| # | 需求 | 推荐方案 | 自建vs外部 | 成本/延迟量级 | 主要风险与降级 |
|---|---|---|---|---|---|
| 1 | 位图→矢量 | **vision-LLM 出草稿 + 人工编辑器描摹**,validate 兜底 | 外部LLM+人在环 | 真实成本含多图/OCR/重试,**远超 $0.02**;5–15s异步 | **最高风险(对抗①)**。随手拍户型图 LLM 出的是拓扑级错误(房间数/连接错),不是坐标微调,编辑器修拓扑错比从零画慢。**降级:LLM 草稿仅作底图描摹参考,用户手工拉矩形;"从零手绘"永远作兜底入口;对外不承诺自动矢量化精度。先时间盒试点验证。** |
| 2 | 尺寸校准 | 现引擎+编辑器,`/derive` 同步 | 已自建 | ~20ms | **真实尺寸需度量锚**:无尺寸文字则需 OCR(独立失败源)。**UX:单基准标定**(指一段已知长度墙输入mm,全图缩放),把 N 次校准降为1–2次锚定(对抗⑪)。 |
| 3 | 矢量→轴测 | 现引擎,已成立 | 已自建 | SVG毫秒+PNG~1s | 受矩形限制约束。 |
| 4 | 风格→软装 | 网关 chat + json_mode → **每房选型+风格(不吐坐标)**;坐标用**人工模板布局**落点 | 外部LLM,布局模板=新建 | <¢/次,秒级 | **"确定性布局求解器"是净新增研究级难题,代码库不存在(对抗⑩/⑭)。MVP用预置模板(房型→2-3套布局),LLM只选模板+替换单品,求解器仅做模板内冲突夹取。** 输出按家具白名单校验拒幻觉。 |
| 5 | 软装调整 | 现编辑器 | 已自建 | — | — |
| 6 | 方案→照片级轴测 | 直连 provider img2img + ControlNet 锁`轴测线稿净版.png` | 外部(fal/Replicate/Gemini) | $0.04–0.08/张×重试,10–30s | **不叫"照片级"(对抗⑤):等轴测=平行投影,与扩散模型透视先验对抗,易出CG/插画质感。定位"高质感效果图"。两个独立指标分别小样验收:风格写实度 + 单品一致性。** |
| 7 | 空房照+方案→效果图 | 直连 provider inpaint + depth;**独立产品赌注** | 外部,Phase2+隔离试点 | 单房5–10次重试,**单套真实数分钟、远超$1–2** | **最难、最贵、无几何护城河(对抗②/④/⑮)。命门=方案一致性:text-prompt 锁不住"那张具体沙发"的 SKU 身份,跨房跨视角必漂。重设承诺为"风格化氛围参考图"非"方案精确可视化",UI文案必须如此。方案级一致需 IP-Adapter/参考图注入,成本高一个量级。masked inpaint 有结构性矛盾(掩膜紧=悬浮假家具/松=改硬装),改用全图编辑+人工挑选。** |

**#1、#7 试点验证法(强制):** 各 1–2 周时间盒。#1:对 10–20 张**真实用户拍摄**样本量化"编辑器还要改几步/几分钟"+OCR命中率;#7:对真实空房照量化成片率与跨房一致性指标。**验证通过再产品化,失败维持人工现状,不污染主架构。**

**单位经济(对抗⑥/⑳):** 成本按**平均重试次数**建模而非单次成功。内容哈希缓存**只对确定性 SVG→PNG 有效,对 #6/#7 生成式步骤几乎无效**(每个新户型都是首跑)。必上:每用户配额/积分 + 接口限流 + 单户预算硬上限 + 低清预览先行/高清按需。定价 ≥ 直接AI成本×含重试安全系数。

**合规前置(对抗㉑,五视角全漏):** #7 上传用户家中实拍照到境外 provider,触发 PIPL 告知同意/数据出境/留存删除义务 + provider 训练复用条款。**这是 #7 合规上线的 gating 条件**:用完即删短TTL、选可关训练复用的服务、评估境内推理(规模化自建 inpaint 的另一驱动力)。

## ⑤ 分阶段迁移路线

**Phase 0 — 去硬编码 + 内核库化(1–2 周,纯重构)**
- **第一步不是搬文件,是去硬编码绝对路径(对抗㉛/㉗):** svg2geometry.py 的 `SRC_DEFAULT`/`OUT_DEFAULT`、build.py/serve.py 的项目根路径与 `HOUSE="D"` 全部改相对/配置。**否则一搬文件金测先挂,谈何兜底。** 路径清零是显式交付物。
- 抽 `floorplan_core` 包;`render()` 改返回字符串;中文模块名→英文。
- verify_golden 升 pytest 进 CI;**补家具渲染快照测试**(几何 derive 有金测可放心,家具/MODELS 渲染无金测,改前必补,对抗㉖)。
- `git init` + `.gitignore`。风险:低(几何侧),中(家具/渲染侧需先补测)。

**Phase 0.5 — 商业与可行性验证(对抗⑫,价值优先于工程)**
- 极薄内部工具(单户型、无鉴权、文件存储),对 **3–5 个真实意向/付费客户手工交付 #1 和 #7**,量化真实成片率与付费意愿。
- **正交可行性抽样(对抗③/⑬/㉘):** 对目标楼盘户型库统计"近似正交矩形可表达率",用数据决定:(a)限定矩形户型细分市场,还是(b)把 geometry 升级为 polygon/任意角(=核心算法重写,**独立高风险里程碑,不能藏在"引擎是资产保持不动"里**)。schema_version 预留 v2 rect 与 v3 polygon 并存。
- **不通过则不上多租户平台。**

**Phase 1 — 单用户 MVP(3–5 周,按单人产能校准,对抗㉓)**
- FastAPI 单体同进程 import 引擎 + 1 个 RQ worker + SQLite/文件存储 + 托管鉴权(或暂无)。
- React 移植编辑器,多户型多项目。
- 覆盖 **#2/#3/#5 完整,#4(网关chat+模板布局),#6(直连provider img2img)**。
- **#1 以"AI草稿+编辑器描摹+从零手绘兜底"形态上线,不进关键路径。**
- 数据债 1/2/3 在此前修复(verify_golden + 新增渲染快照兜底)。
- 风险:编辑器移植量、#6 一致性(seed/参数模板缓解)、容器内 CJK 字体。

**Phase 1.5 — #1/#7 时间盒 spike(与 P1 并行,不进主干)**

**Phase 2 — 规模化(有重复付费客户后才做)**
