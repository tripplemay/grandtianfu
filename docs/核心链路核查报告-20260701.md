## 三、已实现功能的 UX 优化点

### 中等(建议尽快)
1. **方案对比列头「家具 N·效果图 N」恒显 0**(`compare/page.tsx:77-78` 硬编码),与方案中心真实数字矛盾 —— 用已取的 `listRenders` 直接填。
2. **「方案对比」侧栏挂「即将」徽章却已上线可用**(`studioRoutes.tsx:56-61` comingSoon=true)—— 去掉;`§14.18` 的「下一阶段」只针对空房/实拍。
3. **AI「基于方案」下拉默认 `default` 在新户型版本下悬空**,提交必被后端拒 —— 初值改 `schemes[0]?.id`,无方案时禁用生成按钮。
4. **归档方案成终态**,误归档后无 unarchive/复制/删除补救 —— 提供 unarchive 或放行 archived 复制。
5. **「继续调整」成功后只刷列表不跳转**新草稿编辑器(`§20` 要求跳转)—— `adjustScheme` 后 `router.push(.../editor?scheme=newId)`。
6. **超尺寸家具在轴测被静默内缩/移位,平面图仍显原尺寸**,两图不一致且无提示 —— 把 `scene.validation.adjustments` 明细透传前端提示「N 件已自动调整」。
7. **切换方案整块重载**,同 baseline 几何未变也重派生,画布闪回「加载中」并重置视口 —— baseline 未变时只重载 furniture。
8. **LLM 返回候选少于请求数 / 空候选静默落盘**,用户看到 0 家具方案却无解释 —— 追加 warning。

### 低(打磨)
9. 顶栏搜索框、顶/侧栏「退出」为**死控件**(旧审查 MEDIUM-04 未修)—— MVP 阶段移除或禁用。
10. `ConfirmDialog` 破坏性二次确认**缺 Escape/焦点管理**(旧审查 LOW-02)。
11. 只读家具 Tab 保存把「只读」误报成「加载失败」;概览户型卡不按草稿态切文案(`§8.1`)。

## 四、确认 Bug(数据安全 / 越权优先)

### 🔴 高危
- **`validate` 路由缺 `GEOM_READONLY` 且触发落盘自动迁移**——唯一绕过只读护栏污染 live 数据的写路径(`main.py:507-512`;`validate_baseline→_ensure_project_structure→migrate_project(dry_run=False)` 真实建目录 + 恒写 `validation.json`)。
- **editor 只读判定绕过 `context.isHistorical`**——直链 `?scheme=<历史/未知方案>` 或 context 加载中时,编辑器**以可写模式挂载**,踩「已确认只读」红线(`editor/page.tsx:27-29`)。
- **避门端到端落空**——门几何在 `room_brief` 算出并标注「供避让」,却从不传入 `layout`(`furnish.py:121` 只传 room_id+items),大件 footprint 可挡门。

### 🟡 中危(确认)
- render 页完全不消费 `isHistorical`(`render/page.tsx:179/191`),历史版本方案仍可触发 AI 生成(`§14.17` 越权)。
- confirm 临界区跨多文件非崩溃原子(`baselines.py:628/633/643`),崩溃留「楔死」状态并使 legacy 写门禁(`main.py:823` 仅 `confirmed` 才拒写)在 `superseded` 下失效,根 geometry 可被旧接口覆盖。
- 几何草稿键未含 `baselineVersionId`(`draft.ts:15`)+ undo 历史栈未按 scheme/baseline 隔离 → **跨版本数据污染**。
- AI 落位:**家具重叠**(实测真实客厅 sofa/coffee_table footprint 相交)、**放不下的件静默丢弃无 warning**(`layout.py:77-86`)。
- 圆形件/绝对件跳过归一化却仍被墙碰撞校验硬阻断 AI,无自愈(`scene.py:511`)。
- D-only 列表未过滤,点非 D 项目静态导出 404(旧审查 MEDIUM-02 未修)。
- 并发:`create_project` 同名 rmtree 竞态删他人目录;`project_lock` 无破锁 kill -9 后永久阻塞;`append_render`/`set_preferred` 无锁 RMW 丢历史/破坏首选唯一性。
- **furnish 完全绕过预算**(`main.py:884-953`):不 reserve 不计 token,LLM 成本无配额且 `/api/ai/status` 严重低估。
- 归档/历史方案被写门禁过度限制,垃圾方案无法软删(`schemes.py:585-586`)。
- 移动端 `pointercancel` 不清捏合状态,残留幽灵触点(旧审查 MEDIUM-05)。
