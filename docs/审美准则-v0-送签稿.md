## ① 一句话品味主张（POV）—— 这一节的灵魂是空的，只有你能填

**先说机器里确实收敛出了什么（可反推的部分）：**

- **空间真值不可让渡。** 按图落位、保留隔墙、门窗不漂移、"只把平面材质化、绝不移动/新增/删除/合并任何东西"——**这是全稿唯一真正被机器强制的取向**。**[证据](代码强制)**（`prompt_gen.py:206–212` 的 `KEEP EXACTLY…do NOT move/add/remove/merge`；`eval_harness.py:87` 的 `UNACCEPTABLE_KEYS`）

**然后是空槽——这才是本节的重点：**

原稿把"轻奢不是豪华的缩小版，而是豪华的反面"盖了 [证据] 章。**撤回：这句话全仓根本不存在，是起草人自己造的金句。** 证据只支持"低饱和/重材质/克制"，**不授权**"豪华的反面"这个断言。它是一个**候选主张**，不是既成事实。

**[推测·待确认]** 机器里有材质表和禁令，唯独没有**"为谁、造一种什么生活情绪"**这一层。这句必须你亲手写。给三个方向（择一或改写，**别都要**——选哪个决定后面所有点缀往哪偏）：

## ② 配色原则

### 与硬装的关系：顺色、延展、不跳脱
机器在留空风格时，一律回退到暖白胡桃 + 石材 + 黄铜的默认 palette。**[证据](代码强制)**（`prompt_gen.py:216–217`，注意：这是**回退兜底**，用户一旦指定风格就让位）"软装不可跳脱硬装暖调"作为**规则**只在设计稿。**[证据](设计稿)**

### 控色比例：**70 / 25 / 5**（刻意不用教科书 60/30/10）
**[证据](设计稿)**（`阅天府D户型…方案.md:51–53`）—— 这是全稿**最硬的两枚味觉指纹之一**（详见 ⑦）：把点缀权重从通用的 10% 再压到 5%，本身就是一次"更克制"的立场表态。请沿用，别改回 60/30/10。
- **主色 70%**：暖白 / 燕麦米 / 浅灰咖（墙顶大面、大件布艺）
- **辅色 25%**：胡桃木棕 / 焦糖驼色 / 雾霾灰蓝（家具、皮革、单椅）
- **点缀 5%**：香槟金 / 古铜 / 墨绿 / 酒红（金属、抱枕、艺术品、绿植）

> **订正原稿两处越界**：（i）原稿称"70/25/5 是写死的兜底逻辑"——**假的。代码里写死的是 palette 字符串（walnut/dark-green velvet/brass），比例根本不在代码里。** 70/25/5 是设计稿约定，不是机器逻辑。（ii）原稿称"两处硬编码兜底"——实为**一处代码 fallback + 一句描述它的注释**，凑数了。

### 签名重色（墨绿 / 黄铜）：小面积、定点
D 方案确实把一把墨绿旋转椅放在了客厅景观角。**[证据](数据)** —— 但原稿由此推出"墨绿椅只属景观区、**不得散落他处**"并盖 [证据] 章，**是从单个落位实例外推出一条禁令**。一个数据点 ≠ 一条声明的规则。

**[推测·待确认]** "签名重色是否锁定专属区、锁哪几处"——需要你把它从**一次摆放**升格为**一条规矩**，才算数。

### 冷暖、明度与金属
整体暖调、低饱和；灯光 2700–3000K 暖间接光。**[证据](设计稿)**

> **订正"忌冷不锈钢"**：原稿称金属"忌冷不锈钢、冷金属被明令排除"并盖 [证据] 章。**核对结果：这句只是设计稿一行话（`:17`），机器零强制；而且同一份设计稿 `:56` 把"香槟金不锈钢"列为认可材质。这两条自相矛盾，原稿把矛盾抹掉了。** 所以"忌冷不锈钢"目前**不是**已确立的规则——

**[推测·待确认]** 你得先裁决这个矛盾：**是"金属只准暖色、不锈钢一律不用"，还是"暖色不锈钢可用、只忌明晃晃的冷不锈钢"？** 定了才配写进准则、接进 brief 负词。

**[推测·待确认]** 还有三个色彩容差是空槽，只有你能钉死：低饱和到什么程度算"寡淡"？墙面明度上限是多少（会不会白到发飘像样板间）？**"雾霾灰蓝"这个唯一的冷调辅色（`:52`）到底留还是砍**——它和"全屋暖调"本身就打架，机器里没有裁决。

## ③ 材质原则

### 三件套语汇（这一格是设计稿与代码少有的重合处）
核心三元组 **胡桃木 + 洞石（米黄）+ 微水泥**，配白大理石台面、暖橡木地板；软装侧丝绒 / 半哑光真皮 / 羊毛真丝地毯 / 亚麻棉麻窗纱 / 黄铜五金 / 藤编陶艺点缀。**[证据](设计稿 + 代码强制)** —— 难得：代码的默认 palette 里确实含 walnut / travertine / microcement / marble / caramel leather / brass / cream textiles（`prompt_gen.py:216–217`），是全稿**唯一**设计稿主张真被写进机器的材质表（虽然只是提示词建议、非硬拦截）。

> **删除三条"每家都说"的稀释剂**：原稿的"呼应+层次对比、忌同质堆砌"（装修公司话术最大公约数）、"每房设一处焦点墙"（样板间导购公式）、"窗帘一律纱+遮光双层"（中高端住宅规格默认项，不是品味选择）——这些反过来说没有任何工作室会认领（"我们不要层次""我们爱同质堆砌"），所以是零信息。**唯一值得留的是把"焦点墙"升级成一条带脾气的禁令**（见下）。

**[推测·待确认]** "焦点墙"要立成规矩，就得是：**每房焦点 = 1 且仅 1，禁一室双主角。** 设计稿只说了"每房有一处焦点"，"仅一处、禁双主角"这半句是空的，你来定。

**[推测·待确认]** 现有信号能答"用哪些材质"，答不了**"哪两种放一起是我们的签名对撞、哪种组合是禁忌"**。例：胡桃木 + 墨绿丝绒是招牌，那胡桃木 + 大理石够不够、要不要一道皮革或黄铜收口？"忌同质堆砌"是原则，但**全木、全石算不算堆砌**没有清单。这份**材质配对表**是你的手艺，机器给不出。

## ⑤ 克制 / 留白 / 可落地原则

**这一节里，真被制度化的克制是这三条（其余请当参考，不当战绩）：**

- **硬装优先、软装换皮**：换风格 = 换可移动软装/家具材质/窗帘/地毯，**绝不改墙体、门窗、固定硬装**。落成 brief 字段 `keep_hardscape` / `banned_materials` / `banned_colors`——**克制在这里不是口号，是数据结构。[证据](代码强制)**
- **功能忠实、不重新诠释空间**：厨房不变书房、阳台是洗烘不是客厅、卧室不改花园；书房与次卧二保留隔墙。**[证据](代码强制)**
- **秩序化点缀**：签名单品与重色限定区域、每房一焦点、追求呼应而非满室对比。**[证据](设计稿)**

> **删两条空话**：原稿"把自然光/景观当设计要素……南向落地窗、临窗休闲角"（每一家都会写，没人会说自己不重视自然光）、"预算压在天天接触的核心件上"（标准 value-engineering，家装顾问人人会讲）——零信息，删。

**[推测·待确认]** 关于"留白"有一处**内部张力，机器里两种声音打架**：一条线主张"留白/低密度/米白暖调"（提示词与预设层）；另一条线——**D 户型实际落地了 55 件家具、8 座餐桌、成套卧室、多柜多绿植**（数据）——是"生活化偏满"而非性冷淡极简。**到底是"留白派"还是"烟火气派"？** 这决定软装的"满度阈值"，是全稿最需要你一句话定音的空槽之一。

## ⑦ "招牌 vs 合宜"的证据化判断（论据已挤水）

**结论分两层：这大概率是一家"强 POV 招牌"工作室，但（i）它的招牌是"克制"本身，（ii）这枚招牌目前是"声明的"、不是"验收过的"，且（iii）它的 POV 证据只剩两条经得起查。**

**原稿论证"强 POV"用了三条，我逐条核对——两条站不住，撤下：**
