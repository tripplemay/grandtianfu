# calib-cure-b3 F005 — 3D / 简模引导路线评估（go/no-go + b4 建议）

> **性质：研究产物，非产品 signoff。** 执行：Generator（spike 研究码 `scripts/spike/`，不 import `main.py`）。
> 日期：2026-07-20。预算授权：用户 2026-07-20（relay 单后端 4 图）。
> 立项问题（spec §D5 / features.json F005）：**在粗相机 / 无精确标定的前提下，3D / 简模引导能否产出可用实拍图？**
> 起因：b2 L2 判定 solver 修复必要但不充分，正对墙拍照片几何退化 → 想知道能否绕开"精确逐张标定"这条难走的路。

---

## 0. 结论速览

| 问题 | 结论 |
|---|---|
| **3D / 简模引导能否降低对精确标定的依赖？** | **NO-GO。** 引导图 = 世界几何经相机投影；相机误差原样传递，3D 的丰富度与标定精度**正交**，救不了坏标定。 |
| 相机过门时，引导是否被忠实执行？ | **是。** L0/L1 两臂都逐件跟随（沙发/茶几/餐桌椅/边柜/电视柜位置朝向可对上）。几何锁定按设计工作。 |
| 相机崩坏时会发生什么？ | **模型静默丢弃引导、自行编排落位**，产出"好看但不是用户方案"的图。**比崩坏更危险，因为没有任何信号提示失败。** |
| 现有 auto_check 能否发现这种失败？ | **不能，且判反了。** 引导全崩那场 L0 得 **score 1.0 ✅ 通过**；引导良好那场 L1 反而 ❌。详见 §4。 |

**一句话给 b4：** 别再投"用 3D 引导绕开标定"；把钱投在让标定成功（Prong 1）——因为**一旦相机过门，出图质量已经够用**。另外补一个"引导是否真被执行"的保真度检测，现有指标是瞎的。

## 1. 方法

用**真实生产病例照 + 它们自己存着的真实相机**（不合成、不挑好数据）：

| 场景 | 房间 | 照片 | 存档相机实测 | 定位 |
|---|---|---|---|---|
| `guest2_coarse` | r_guest2 书房（3 件家具） | `472015c4…jpg` | **reproj 2353.4px**，hfov 94.7°，相机高 1370mm | b2 L2 那张"标不出来"的正对墙拍照；粗相机压测 |
| `live_leastbad` | m_living 客餐厅组（6 件） | `3548aa76…jpg` | reproj 未记录，hfov 101.2°，相机高 1382mm（**过高度/视场门**） | 生产里**相对最好**的一台相机；对照 |

- 两臂：**L0**（半透明彩盒，产品现状）vs **L1**（部件级不透明 3D 简模）。同 photo / 同 camera / 同 furniture / 同 style，仅换引导表示法（公平性见 `scripts/spike/README.md` §两臂公平性）。
- 后端：**仅 relay**（用户授权范围）。**fal 臂未跑 → 跨后端鲁棒性本批未验证**（b1 spike GO 条件之一，仍悬空）。
- 记账：4/4 成功，relay tokens 合计 **19077**，耗时 139–179s/图。

## 2. 决定性证据（零成本，出图前就已得出）

**干跑产出的引导图本身就回答了立项问题。**

- `guest2_coarse`（坏相机）：书柜悬在半空、穿透天花板与窗墙；椅子浮在窗台上方；书桌歪出画面。**没有一件落在地面上。**
- `live_leastbad`（过门相机）：沙发落地、茶几有腿、餐桌/边柜/电视柜各就各位，透视与照片严丝合缝。**完全可用。**
- **L0 与 L1 在坏相机下崩坏方式一致**（L0 的彩盒同样糊满墙面/悬空/出画）。

→ **失败由相机决定，不由引导表示法决定。** 这是 NO-GO 的直接依据：引导是投影产物，上游相机错了，下游再精细也无意义。

## 3. 出图结果（花钱臂）

| 场景 | 臂 | 落位是否跟随引导 | 观察 |
|---|---|---|---|
| guest2_coarse | L0 | **否** | 输出是一个合理书房，但落位是模型自己编的 |
| guest2_coarse | L1 | **否** | 同上：书柜规整贴右墙落地、椅子坐地、书桌落地——**与崩坏的引导毫无关系** |
| live_leastbad | L0 | **是** | 逐件对应；沙发被拆成两张（引导是单件 L 形转角） |
| live_leastbad | L1 | **是** | 逐件对应；**沙发保持单件 L 形转角，形体更贴引导** |

**两条读数：**

1. **粗相机下模型不产出垃圾，而是产出"漂亮的错图"。** 它把崩坏引导当噪声丢弃，只取语义（有书柜/椅子/书桌），落位按照片自身透视重编。结果是**静默的不忠实**——用户拿到一张好看的图，但那不是他的方案。这正是本项目 2026-07 已明确**下线的"AI 自动落位"**行为，在坏标定下从后门回来了。
2. **L1 vs L0（好相机下）：L1 形体保真度略优**（单件 L 形沙发 vs 拆成两张；茶几比例更贴）。与 b1 F012「L1 GO 带条件」一致，但**这是另一个问题**，不构成本条 spike 的 GO。

## 4. 附带发现：auto_check 指标在此失效且判反

| 场景 | 臂 | score | 自动验收 | 实际 |
|---|---|---|---|---|
| guest2_coarse | L0 | **1.0** | ✅ 通过 | **引导全崩、落位完全不是方案** |
| guest2_coarse | L1 | 0.85 | ❌ 结构改动 | 同样不跟随，但被判失败 |
| live_leastbad | L0 | 1.0 | ✅ 通过 | 忠实跟随 |
| live_leastbad | L1 | **0.917** | ❌ 结构改动 | **忠实跟随且形体最好，却被判失败** |

`evaluate_geometry_lock` 度量的是"盒区内外的结构变动"，**不度量"家具是否落在引导指定的位置"**。引导崩坏时盒区本身就是错的，于是指标退化：最差的一格拿了满分，最好的一格被判失败。

→ **这是一个真实的产品缺口**：坏标定产出的"漂亮错图"会带着 score 1.0 静默通过自动验收。（现行 reproj/高度硬门是唯一拦得住它的东西——门是承重的，不能放宽。）

## 6. go/no-go 与 b4 建议

### NO-GO：不立"3D 引导替代精确标定"批次
理由见 §2：引导是相机的下游，3D 化不改变这个依赖关系。b1 spike 判 L1 GO 是**画质/形体**层面的结论，与"减少标定依赖"无关，本批不应被引用为后者的支持。

### 建议 b4 做三件事（按价值排序）

1. **继续 Prong 1（让标定成功）——已被本批正面证明。** `live_leastbad` 证明：相机一旦过门，出图忠实且质量可用。标定是整条链路的瓶颈，也是唯一的瓶颈。
2. **补"引导保真度"检测（新缺口，§4）。** 现有 auto_check 无法区分"跟随引导"与"丢弃引导自由发挥"。建议：把引导的家具足迹 mask 与出图中的家具区域做重叠度比对，低于阈值即判"引导未被执行"。否则坏标定的漂亮错图会带 score 1.0 静默过关。
3. **存量坏标定清理（§5）。** 生产 7 份全是门前时代 legacy 存量，多份物理不可能。建议给用户一个"重标"入口并把明显不可能的标定主动失效，而不是留着让它们在渲染时才 409。

### 不建议
- 不建议为"粗相机 + 3D 引导"做兜底出图路径：它产出的是静默不忠实的图，比明确失败更伤信任。

## 7. 诚实边界

- **仅 relay 单后端**；fal 臂未跑 → 跨后端鲁棒性未验证（b1 GO 条件之一仍悬空）。
- **n 极小**：2 场景 × 2 臂 × 1 次，无重复采样、无 seed 变化。"模型丢弃坏引导"这一结论来自 **1 个场景 × 2 臂**，方向明确但样本单薄。
- `live_leastbad` 的相机是"**相对最好**"，非经验证的正确相机（生产中不存在后者）。"忠实跟随"是与**引导图**逐件目视比对，**未与 `furniture.json` 坐标做数值对齐**。
- 落位跟随/不跟随为**目视判定**，无量化指标（§4 说明现有指标不可用于此）。
- **PIPL：** 病例照、引导图（含照片像素）、出图产物**一律未入 git**；仅存在本地会话沙箱，随会话销毁。本报告只含文字与数值。
- b2 L2 曾记 r_guest2 "标不出来"，本批实测其存档 `calib=True`——二者不矛盾：存的是门前时代的坏标定（reproj 2353px），正是 b1 evaluator 记录的"渲染 409 reproj2353px"那份。
