# AIGC 回归评测集 (P2-2, 2026-07-11)

审查报告 P2-2 (302-310) 要求"建真实样本集 + 每次改 prompt/provider/geometry-lock/validation 后跑评测,
报告几何适配/落位/材质一致性/失败类型统计/人工可接受率"。本次把评测集补成**两层**,互补:

## 一、确定性回归评测集 (新增, 无需真实生成图)

`apps/api/aigc/eval_scenarios.py` — 批2~6 加的一整套**确定性输入侧检查**(布局 lint、场景校验、
盒子投影可用性、标定生命周期) 都是给定几何+家具即可断言的纯函数。评测集在真实 D 户型上叠加
特定家具构造场景,断言各检查的产出。**改 lint/validation/geometry-lock 后跑一次即知有无回归**。

场景矩阵 (好布局 + 各失败类型):

| 场景 | 失败类型 | 断言 |
| --- | --- | --- |
| good_living / sectional | 好布局 | 零 lint 问题 (含组合沙发拼接不误报碰撞) |
| floating_wall_unit | 悬空柜类 | `LAYOUT_WALL_UNIT_FLOATING` |
| backs_glass_wall | 大件背贴玻璃幕墙 | `LAYOUT_LARGE_BACKS_FULL_WINDOW` |
| furniture_overlap | 家具碰撞 | `LAYOUT_FURNITURE_OVERLAP` |
| dangling_room | 家具挂空房间 | 场景校验 `DANGLING_FURNITURE_ROOM` (ERROR) |

跑法 (本地, CI 不跑 pytest):

```
PYTHONPATH=packages/floorplan_core:apps/api python3 -m pytest apps/api/tests/test_eval_scenarios.py -q
```

`test_all_scenarios_pass_on_real_d` 是回归门 —— 任一确定性检查回归 (漏报/误报) 即失败。
`coverage()` 汇总失败类型覆盖是否完整。`to_markdown()` 出报告表。

## 二、出图结果评测集 (既有, 需 live provider + 部分人工)

`apps/api/aigc/eval_harness.py` + `scripts/run_image2_eval.py` — 评的是**真实出图结果**。
本次把**失败类型统计**与**可接受率**从"纯人工"升级为**自动**:

- `metrics_row` 从 `record.auto_check` (批1落库, 批5语义并入) 自动提取 `auto_check_ok / score /
  failure_types` (失败词 -> 漏画/彩盒残留/重取景/结构改动/材质漂移/盒内类别错; 未知失败词记"其他"不漏统计)。
- `summarize` 自动出 **auto_check 通过率** (代理"人工可接受率",人工评分槽仍可回填) + **失败类型分布**。

跑法 (对着已起的服务, 烧 provider 额度, 不入 CI):

```
python3 apps/api/scripts/run_image2_eval.py   # 详见脚本内说明; 报告落 gitignored artifacts/
```

## 覆盖边界 (如实记录)

- **确定性层已自动、可 CI-外本地门禁**: 布局/几何/校验回归零成本可测。
- **像素/语义层需真实图**: acceptance 启发式的隐性漏画、semantic VLM 的类别/材质判定,须真实出图才能评;
  由第二层承接,其失败类型现已自动分类。
- **真实样本语料受 PIPL 红线约束**: `data/uploads/` gitignored (用户空房照不入库),故跨户型/光照/照片质量的
  真实语料只能走 `config.json` 外挂真实 photo_id (第二层的运行器), 不落仓库。
