## 增补:合并组 AI 落位 · 异形二期 b (2026-07-06)

AI 链路残留清单里唯一还没「当一个房间」处理的环节。P3 已把 merge 组折叠成一条 rep
简报(尺寸=并集 bbox)喂 LLM,但 `layout.plan_report` 对 merge 零感知,只在 rep 单腿
矩形落位 —— 另一条腿永远空置、且 LLM 按并集尺寸选型却塞不进 rep 腿 → 假性「放不下」。

- **并集落位**:`layout` 检测 rep 属组(`merge_groups`)→ 走并集分支 `_place_group`:
  候选按各成员腿墙向内网格 **round-robin 交错**(跨腿铺开,而非填满一腿);每件
  footprint **完全落入某一成员腿**(per-leg `_fits`)—— 既排 L 凹口,又不骑腿间缝
  (骑缝件会被下游 scene 组感知夹取推回单腿、偏离落位);门/窗净空取外墙并集;每件
  按落入的成员腿归属 + **腿内相对坐标** emit —— 与 `scene.rect_of[room_id]` 原点还原、
  `prompt_gen` base_off 成员腿重投影的既有契约一致(无需改 scene/prompt)。方向件
  orient 取最近**外墙**(跳过与相邻/重叠腿相接的内部边,免床头/柜背朝虚拟墙)。
- **共享原语**:四壁开洞匹配 `geometry.edge_openings` + 外墙开洞并集
  `geometry.group_exterior_openings` 抽为单一真源,`room_brief` 与 `layout` 复用,防
  双源漂移;`room_brief` 输出经 SHA-256 验证逐字节不变。外墙判定用**几何**(开洞外侧
  点是否落在另一成员)而非"命中≥2成员"计数 —— 正确处理重叠腿(内部边只命中一次)
  与共线外墙接缝洞(命中两腿但外侧在外)。
- **字节安全(已核实)**:layout 输出只写新 AI 候选方案 `schemes/<scheme_ai_*>/`,
  从不碰定稿 `data/projects/D/furniture.json`;两条 golden(verify_golden 只跑几何、
  test_render_snapshot 读静态 furniture)与 layout 隔离 → **非 golden 锁,普通 PR**。
  非 merge 路径逐字节不变(现有单房 test_layout 全绿);`_door_zones`/`_boxes_intersect`
  签名不动(scene.validate_scene 复用)。
- **测试**:test_layout +5(跨腿分布/成员归属腿内坐标/凹口排除/scene+prompt 往返/
  内部共享边门净空抑制),test_merge_groups +2(edge_openings/group_exterior_openings)。

## 增补:CP5v2 贴合并房点选目标 (2026-07-05)

原 CP5 (PR#14) 的「贴合并房」盲取相邻候选数组第一个 (`cands[0]`,按 rooms 存储序),
四面邻房时无法指定并入目标;且 space 归属为「发起方胜」——选中小空间发起并房会让
邻居大房改姓。本增补重做交互与归属语义,并顺带修三项卫生问题:

- **点选目标**:多候选 -> 画布点选模式 (候选绿色虚线高亮 `MergePickLayer`,点击并入,
  Esc/点空白/再点按钮取消);唯一候选直接并入。状态 `mergePick` 持在 useGeometryEditor,
  画布全部指针入口 (房块/背景/把手/门窗/自由墙/翻转) 均有拦截,与插入模式互斥;
  点选期间隐藏缩放把手 (热区盖共享边)。任何几何写入 (updateG/undo 快照还原) 自动
  退出点选,提交前还按当前几何复验邻接 (候选是进入模式时的快照,防过期误并)。
- **space 归目标房**:`mergeIntoTarget` (lib/floorplan/merge.ts,纯函数) —— source 整组
  并入 target 的 space+merge 组,保留目标名称/类别,并把目标房当前 label 刷新到 space
  标签 (画布组标签取 space 标签,与 toast 一致);「打通」同语义对齐:目标 = Shift+点
  的第二房 (room2,取消选择时同步清掉),框选/全选无 room2 时取面积最大者 (平局最小 id,
  与代表房规则一致);toast 明示「谁并入谁」。并房后主选被并房 —— 紧接「分隔」即还原。
- **prev_space 快照**:被并房间记 `rooms[].prev_space {id,label,category,style}` (引擎
  不读,byte-safe 由 test_prev_space_byte_safety.py 锁死);「分隔」优先按快照还原原
  名称/类别并**复用原 space id** (开洞 between 引用保持一致,不重映射、可逆)。
- **孤儿 space 清理**:并房/分隔后 `pruneOrphanSpaces` 移除无房间且无开洞 between
  引用的 space 条目;被开洞引用的原 space 保留 (分隔还原时直接复用其定义)。
- **合并边界洞 (已知限制)**:共享墙并房后消隐,墙上 cut 门/通道会悬空 —— 引擎 D12
  判定报 **ERROR 拒存** (非 WARN;p01/d02 类均如此)。并房 toast 会检测合并边界上的
  开洞并明确提醒「保存前请处理」,校验条可点击定位;是否自动降级/删除留待后续定案。
- **画布组标签**:编辑器 GroupDecor 组名改取 space 标签 (名称归目标可见);引擎 plan2d
  组标签仍按代表房 (最大面积) —— 引擎侧对齐留待 P5 golden 重冻窗口。
- **安全网**:e2e merge.spec.ts 4 条 (点选并入+space 归属断言/Esc 取消/打通方向/分隔
  prev_space 还原,基于 D 户型 r_garden 入户花园恰有 2 候选);引擎 3 条 prev_space
  字节安全 (derive/plan2d/validate)。

## 增补:CP5v3 合并组单房间视图 + 三项 UX 修复 (2026-07-05)

用户定案:合并组在编辑器中应呈现为**同一个房间**(旧房间不再以独立身份示人),多边形
房间路线维持否决(引擎地基不动,组=矩形成员+逻辑房)。本增补落地该语义 + 修三个
生产暴露的 UX 缺口:

- **单房间组视图**:侧栏点组任意成员 → 组级面板(组名=space 标签/类型/并集面积
  精确矩形并集 groupUnionArea,重叠不双计),改名/改类型作用全组;成员矩形/墙面
  材质/删除收进「高级」折叠(含成员切换 chips);混合 type 组(CP5v2 旧数据)给
  ⚠提示+一键统一;「分隔」仍作用当前成员。
- **并房全组统一 type/label**:mergeIntoTarget 把被并房与目标组既有成员的 type
  与 label.zh 统一为目标房(编辑器地板色与引擎出图/简报/提示词全链一致,代表房
  按面积易主也不漂移);原值记 `rooms[].prev_type/prev_label`(引擎不读,字节安全
  测试锁死),「分隔」还原。规则:任何显式 type 修改(单房/组)清 prev_type ——
  显式指定即新真值,分隔不再回滚。组改名写 space 标签+全部成员 label。
- **方案几何页版本管理只读**:判据与后端 409 门同源(project.created_at 非空 ⇔
  project.json 存在;baselines 列表对遗留项目会合成 v1 不可作判据)。只读时几何
  画布交互/键盘改动/几何草稿(autosave 读写与恢复)全部关闭,仅留视口缩放平移;
  Ctrl+S toast 直接给指引原因。家具页不受影响(FloorplanEditor geometryReadOnly)。
- **保存家具校验分级**:RAW 贴墙相交(仅可归一化件:有 room_id 的矩形件)与超高
  提示折叠为「N 项将由渲染自动修正」;圆形件/绝对件的墙碰撞不可自愈保持平铺;
  挡门/越界/中心不在房/ERROR 平铺可点击定位。
- **安全网**:merge e2e 迁到草稿版本上下文(POST /baselines 动态建草稿),新增组视图
  断言 + 方案几何页只读用例(共 5 条);prev_type/prev_label 并入引擎字节安全测试。
  17-agent 对抗审查 12 项确认全修(组名漂移/只读画布漏洞/草稿恢复漏洞/autoWarns
  误标等)。
