# 户型图 → 真 3D 房间模型：可行性与出图质变评估（2026-07-17）

> 问题：① 能否根据现有户型图生成真正的房间 3D 模型？② 有了 3D 模型，出图效果是否有质的提升？
> 关联：`docs/标定功能缺陷核查-20260717.md` §4（根治方案以 3D 壳为对齐靶）、`docs/AIGC链路核查-带评论效果图根因-20260717.md`。
> 结论先行：**① 能——数据已具备 80-90%，本质是"补高度字段 + 升级渲染器"，不是从零建模；② 对"落位与形体"是质变（与已实证的 mask→彩盒梯度同向的下一步），对"写实度"不是（那本来就是 AI 的活）。同一笔投资同时买到标定根治的对齐靶和验收的 ground truth，这是它真正的价值密度所在。**

## 1. 问一：能不能？—— 数据与代码盘点

### 已有（比直觉多）

| 资产 | 现状 | 对 3D 的意义 |
|---|---|---|
| 房间/墙 | rects + merge 组 + free_walls，壁厚分级（外 240/内 140…），mm 精度 | 壳体平面完备 |
| 开口 | 25 个门窗，`wall{axis,at,span}` 毫米级定位 + door_type/hinge/swing | 位置完备，**缺高度** |
| 家具 | 参数盒 `{t,w,h,z,dx,dy,orient}` + 真实毫米高度表（perspective `_DEFAULT_HEIGHT_MM`） | 占位体完备 |
| **部件分解** | **catalog 48 类已带 `plan2d_spec` 声明式外形**（sofa=靠背 0.22+扶手 0.85×0.11；本为 2D 俯视细节设计） | **参数化 3D 简模 = 给现有部件加高度字段**，非从零建模 |
| 微型软渲染器 | `annotate_boxes`/`_box_polys`：盒面投影 + 画家算法排序 + 近平面裁剪，纯 numpy/PIL | **升级为着色不透明简模渲染是同一代码路径的增量**，无新依赖 |
| 相机 | 标定位姿 (K,R,t) + `Camera.project` | 照片视角渲染即插即用 |

### 缺口（可枚举，均为数据补充级）

1. **开口高度**：门高（默认 2100/2200）、窗台/窗楣（南窗全落地=0/2400 已知；普通窗默认 900/2400），补 per-opening 可覆盖字段即可；
2. **真实层高数据化**：现在 2700 硬编码在 `perspective._REAL_CEILING_MM`，geometry.meta 里的 1450 是轴测压扁世界——3D 模型必须引入项目级 `real_ceiling_mm` 字段，终结硬编码；
3. **部件高度**：plan2d_spec 各部件加 z 区间（sofa: 座 420/靠背 800/扶手 620），~20 个常用类型一个下午的数据活；
4. 可选：梁/吊顶/踢脚线——**edit 范式下不需要**（见 §2.1）。

### LOD 阶梯与体量

| 级 | 内容 | 体量 | 停在哪 |
|---|---|---|---|
| L0（现状） | 半透明彩盒 | — | — |
| **L1（甜点位）** | 壳体（墙/地/开口洞）+ 部件级家具简模，不透明、平光着色、同类色区分，照片视角渲染 | 数据补充 + 渲染器升级，**1-2 个批次** | **推荐停这里** |
| L2 | 材质族上色（style_snapshot 驱动）+ 简单光照方向估计 | +1 批次 | 按 A/B 结果决定 |
| L3 | 真资产库 + PBR 渲染器（Blender/three.js 服务端） | 月级 + 部署重 | **不做**——传统 archviz 路线，与"AI 负责写实"的产品前提冲突 |

## 2. 问二：会不会质变？—— 按机制分析

### 2.1 关键架构事实：edit 范式下，房间壳来自照片本身

第7步是**双图指令编辑**（空房照 + 引导图），背景（墙/顶/地/吊顶/灯带）由"keep image 1 unchanged"约束保留——**3D 模型不需要重建房间的写实外观，只需要提供家具的几何真值和用于对齐/遮挡的壳**。这把"真 3D 模型"的工程量砍掉了大半：吊顶造型、材质、灯带都不用建。

### 2.2 质变的机制：模型的任务从"想象"降级为"重绘"

现状：模型看到半透明彩盒 + 文字（"purple box = a full-height bookshelf"），需要**在盒子里凭空想象**书柜的透视、结构、比例——想象错了（书柜画成窄条、餐桌缩进背景）没有任何像素级约束。

L1 后：引导图里是**逐像素透视正确**的灰体书柜（框体+隔板+柜门部件），模型的任务变成"把这个简模重绘成写实家具"——透视、遮挡、朝向、比例、多件关系（椅子环绕餐桌、茶几在沙发前）全部由渲染保证，AI 只负责材质光影细节。

**既证梯度外推**：本仓自己的实验史已经证明这个方向的斜率——平面 mask（只画矮物）→ 3D 彩盒（PR#70，"整屋家具立体成形"的质变）。L1 简模是同一梯度的下一步，且是终点（再往上是 L3 传统渲染，收益转负）。

具体消灭/压缩的失败模式：

| 失败模式 | 现状 | L1 后 |
|---|---|---|
| 书柜"铺满东墙"画成一窄条 | 盒内自由想象 | 简模有隔板/通高框体，宽度像素锁定 |
| 餐桌缩进背景/比例漂移 | near 话术软约束 | 前景大小即渲染大小 |
| L 形沙发两段被并成一张/朝向错 | 两盒+count 文字 | 部件简模天然表达 L 形与朝向 |
| 椅子数量/环绕关系错 | "8 seats" 文字 | 逐把椅子渲染在位 |
| 家具间遮挡错乱 | 画家算法只管盒面 | 全场景 z 排序，含墙体遮挡 |

### 2.3 验收的结构性升级（常被忽略的一半价值）

有了 L1 渲染即有 **ground truth**：逐件精确轮廓掩膜、深度序、期望边缘图。当前验收的最大盲区——"家具画到别处去了"（f4dab9 沙发/酒柜，auto_check 0.967 满分放行）——变成**可量化检测**：输出图在 GT 掩膜内应有该件家具的边缘/纹理响应，掩膜外不应出现同类大物体。启发式即可拦截大位移，不必依赖 VLM；VLM 语义层降级为细粒度兜底。

### 2.4 不会改善什么 + 新风险（诚实边界）

- **写实度/材质丰富度不变**：那是 AI 的活，本来就不弱；
- **服从性要 A/B 实测**：gpt-image-2/nano-banana 对简模引导的跟随度是经验问题。本仓已有教训（relay 复评：**淘汰结论绑定引导方式，换引导要重赛**）——用现成 `eval_harness` + 回归评测集跑 L0 vs L1 双后端对照，量化落位改善后再全量；
- **新失败模式**：简模穿帮（AI 保留灰体质感）——prompt 侧"replace primitive furniture with photorealistic..."+ 引导图 alpha 混合可调；A/B 中观察；
- **对坏标定无免疫**：位姿错 → 简模渲染同样错位。**3D 不救标定；反之标定救 3D**——所以标定根治（缺陷核查 §4）是本路线的前置，两者共享壳体模型。

## 3. 协同全景：一个模型，四处回报

```
真实毫米世界 3D 场景模型 (壳体 + 部件家具)
 ├─ 标定: 对齐靶 (线框叠照片, 特征点自带世界坐标)   ← 缺陷核查 §4 路线一
 ├─ 出图: L1 渲染引导 (替换彩盒, 落位/形体质变)      ← 本文 §2
 ├─ 验收: GT 掩膜/深度序 (落位错从不可检到可量化)    ← 本文 §2.3
 └─ 架构: 统一"两个 z 世界" (axon=压扁视图, 盒=退化视图, 引导=着色视图
     ——同一数据 N 种投影, 根除"借常数"类 bug; 轴测 golden 初期不动, 增量收编)
```

## 4. 路线图建议

| 期 | 内容 | 前置 |
|---|---|---|
| S0 **spike（先做这个）** | 手工搭 1-2 个场景（D 客厅+书房）的 L1 简模引导图，relay+fal 各跑既有 eval 集，量化落位/形体改善 → **用数据决定是否立项** | 无 |
| S1 | 数据补齐：开口高度字段、real_ceiling_mm、部件 z 区间（~20 类） | S0 通过 |
| S2 | 渲染器升级：`_box_polys` → 部件网格 + 平光着色 + 壳体遮挡；新 `form_guidance:"render-guide"` 挂 flag 与彩盒并行 A/B | S1 |
| S3 | 验收升级：GT 掩膜位移检测进 acceptance；与标定根治 C 期（特征点对齐）共享壳体 | S2 |
