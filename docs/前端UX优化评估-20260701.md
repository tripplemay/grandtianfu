# 阅天府前端 UX 优化评估

- 评估日期:2026-07-01
- 范围:`apps/web` studio 工作台全前端(工作流顺畅度 / 页面·菜单·交互合理性 / 与主流工作台差异)
- 方式:7 维多智能体并行评估(工作流走查 / IA 导航 / 三组页面交互 / 启发式心流 / 竞品对标)→ 综合。均基于实际代码,附 `file:line`;竞品项为对标/启发式判断。
- 基准:`docs/项目基线与多方案工作流-产品交互规格.md` §7 主工作流 / §8 页面规格 / §9 文案

---

## 一句话结论

工作流在规格上清晰、功能齐全,但 **UI 从不「推着用户往前走」**——每一步做完都是死胡同,靠侧栏自己找路;缺进度指示与「下一步」主 CTA;长 AI 任务是黑箱;导航层还叠着两套冗余上下文头 + 英文状态直出。本应一气呵成的线性流程,被切成「每步都要回侧栏重新找路」的碎片体验。

---

## 问题 1 · 用户按预定工作流推进是否顺畅?

**判断:不顺畅。** 规格是清晰的线性 8 步,但每个工具型页面完成动作后都是死胡同(只 toast 把用户留原地),三个解锁下一阶段的里程碑(确认户型/确认方案/迁移)成功后只 toast+reload 不放行,再叠加全程无进度指示和状态驱动的「下一步」主 CTA。

| 断点 | 严重 | 现状 → 改进 |
|---|:--:|---|
| 三大里程碑跃迁不放行 | 高 | 确认户型/确认方案/迁移成功后只 toast+reload。确认户型弹窗写着「确认后才能创建方案」,却不带用户去方案中心(baseline/page.tsx:66-71、scheme/page.tsx:221-224/297-316)→ 成功后 router.push 到该去页 |
| 每个工具页是前进死胡同 | 高 | 家具编辑器保存后无「预览/出图」出口、方案预览纯展示、AI 效果图出图后无「设首选/去对比/返回」(useFurnitureEditor.ts:745、gallery/page.tsx:58-97、render/page.tsx:255-278)→ 各页补携带 ?scheme= 的前进 CTA |
| AI 生成是长阻塞黑箱、离页失联 | 高 | 候选 90s、效果图 6min 只显「生成中…」,generating 局部 state 切页即失联(scheme:44、render:32/142)→ 骨架+进度+取消,任务上提项目级(job_id)可恢复 |
| 确认户型「晚失败」 | 高 | 草稿页不展示 validation_issues、有 ERROR 也能点确认,等后端拒绝才知道(baseline/page.tsx:116-144)→ 页内展示校验+可点定位,ERROR 时禁用确认 |
| 确认户型文案虚假吓人 | 高 | 首次确认 V1 硬编码 §9.3「原版本及方案进入历史」+danger 红,此时根本无旧版本(baseline/page.tsx:56-63)→ 按 currentBaseline 分支文案 |
| 创建空方案后不跳编辑器 | 中 | 创建完要回列表重新找卡再点编辑(createBaseline 却有跳转,不一致)(scheme/page.tsx:156-166)→ 成功后直达新方案编辑器 |
| 侧栏「方案对比」死循环入口 | 中 | 常驻入口直点必落「请选择方案」空态、唯一按钮把你原路弹回(compare/page.tsx:130-143)→ 空态就地列方案勾选 |

## 问题 2 · 页面/菜单/交互是否合理、是否符合心流?

**判断:功能齐全,但心流被反复打断。**

| 区域 | 严重 | 问题 → 改进 |
|---|:--:|---|
| 双上下文头冗余 | 高 | 项目页顶部同屏叠「顶栏面包屑 + 粘性上下文头 + 双大标题」,两套 Provider 各自闪烁/项目名不一致(StudioNavbar.tsx:46、layout.tsx:26)→ 合并为单一权威上下文头 |
| 双 active 高亮 | 高 | 「项目台」用 includes('/projects') 判活跃,进内页与「项目概览」同时高亮(Links.tsx:26)→ 精确匹配 |
| 英文枚举直出 | 高 | draft/confirmed/superseded/archived 在 6+ 页直出,overview 还用 '?? confirmed' 兜底 → 统一中文徽章映射(已有 LoadStateBadge 可照搬) |
| 方案卡按钮墙 | 高 | 单卡平铺 8-9 个等权灰底按钮,高频「编辑」与破坏性「删除」同权重(scheme:641-748)→ 主操作+对比勾选常显,其余进「⋮」溢出菜单 |
| 编辑器切 Tab 丢视口 / 撤销无按钮 | 高 | 几何↔家具切 Tab 缩放平移全丢、撤销重做只有键盘(FloorplanEditor.tsx:322-356)→ 共享视口 + 常驻撤销/重做按钮 |
| 只读态仍渲染可编辑外壳 | 中 | 只读方案照样能点保存/加房间,点后被静默吞或误报「加载失败」→ readOnly 一路透传隐藏工具 |
| 外壳死控件 | 中 | 假搜索框、两处失效「退出」、空用户菜单占最显眼区(StudioNavbar、StudioSidebar:280-286)→ 未接入前隐藏 |
| 折叠态侧栏重复图标 | 低 | mini 态「项目台/户型基线」「方案预览/实拍效果图」各共用同一图标无标签兜底 → 换可区分图标+tooltip |

## 问题 3 · 与主流工作台软件的差异

**判断:差距集中在成熟行业基线能力,非有意差异化,尚未补齐。**(AI 场景校验阻断出图是我方领域优势,保留并打磨成闭环。)

| 主题 | 主流做法(Figma/Canva/酷家乐/Notion/Midjourney) | 我方现状 | 建议 |
|---|---|---|---|
| 线性流程进度 | 关键流程放 stepper/进度条 | 只有层级面包屑,9 个并列入口无顺序感 | 加水平 stepper 可点跳转 |
| 状态驱动下一步 CTA | 首页按完成度突出唯一下一步 | 概览三张等权卡,主色还给错了(overview:63-89) | 概览顶部加状态驱动主 CTA |
| 长 AI 任务 | 进度%/可取消/后台跑可恢复/结果高亮 | 只有「生成中…」,离页失联,文案与 timeout 矛盾 | 进度+取消+骨架+项目级可恢复 |
| 画布撤销/缩放 | 常驻撤销(置灰)、缩放 ±/百分比、滚轮即缩放 | 撤销只有键盘、缩放两档跳变、Ctrl+滚轮无提示 | 补常驻按钮与 ± 缩放 |
| 快捷键可发现 | 「?」速查 / 命令面板 | 有成体系快捷键但无处可查 | 加「?」速查弹层 |
| 多方案对比 | 并排同步 zoom/pan | 各列静态图,「相同缩放」空洞满足(compare:213-249) | 三列联动缩放或同步 lightbox |
| 业务状态本地化 | 一律本地化徽章 | 英文枚举直出 | 统一中文徽章 |
| 卡片操作溢出菜单 | 1 主+1 次,其余「⋮」 | 平铺 8-9 个等权 | 溢出菜单 |
| AI 场景校验阻断(我方优势) | 多数出图工具不做结构校验 | render 页 sceneBlocked 拦截保质量 | 保留,补「去调整家具」闭环 |

## 执行建议

先打一批 **quick-win**(改动小、当天可见、直接提升顺畅度与专业感),再啃 P0 两块硬骨头(stepper + 全链路「下一步」CTA、AI 生成进度/可恢复)。

> 方法学备注:竞品对标专项 agent 本次因结构化输出重试超限失败,q3 由工作流/启发式两维的对标发现 + 综合 agent 的产品知识补齐,结论与其余维度同源可信。
