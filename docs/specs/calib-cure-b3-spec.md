## 1. 背景与目标

**L2 根本启示：手工逐张标定本质脆弱。** 即便解算器修好：(a) 正对一面墙拍的照片（常见）几何上就标不出（特征共面）；(b) 即便角落机位，用户还得正确完成"标签↔像素"对应（天花板角难分辨、易镜像）。

**用户裁决（2026-07-19）：两条腿。** 近期用引导 + 早拦解锁用户；同时评估 b1 spike 已 GO 的 3D/简模引导路线能否长期少依赖精确标定。

**目标：**
- **Prong 1（近期解锁）**：让用户**不再白点半天**——退化照片标定前就明确"这样拍标不了，请从房间角落重拍"；能标的照片（角落机位）顺畅标成；窗几何失配不再坑人。
- **Prong 2（评估）**：产出 3D/简模引导路线的 **go/no-go + b4 建议**（在不依赖精确逐张标定前提下能否产出可用实拍图），含**标定不了的照片**（如 r_guest2 正对墙拍）。

**非目标：** 本批不重写渲染管线、不训练/接入新 CV/ML 相机估计模型（若 Prong 2 判 go 再另立 b4）；不改 b2 已验的解算内核。

## 2. 关键设计决策（方向已定，具体实装含开工前调查）

### D1 退化 = 具体可行动诊断，不是泛泛"reproj 过高"（Prong 1 核心）
现 `assess` 对退化只报"reproj 超阈值"；`degeneracy_reason`（b2 F004）报"近共线/单一高度"。b3 升级：**识别"正对墙 / 所选点共面"的退化模式**（3D SVD s3/s1≈0 = 点共面；结合相机解出高度/hfov 极端），给**拍摄级可行动提示**：『这张像是正对一面墙拍的，标定需要角落机位——请站到房间角落、让画面同时带到两面相邻墙 + 地面纵深后重拍』。**门不放宽**（退化照片本就标不准，早拦比出歪图诚实）。

### D2 拍摄 / 构图引导 UI（Prong 1）
在实拍照**上传入口 + 标定入口**加拍摄指南（文字 + 简示意）：角落机位 / 两面墙 / 地面+天花板转角入画 / 避免正对单面墙。复用设计系统组件。**开工前调查**：现有上传/实拍页引导位（b2 F006 已加标定内 NoticeBanner，避免重复）。

### D3 特征供给策略：结构角优先、窗特征稳健化（Prong 1）
b2 F002 派生了窗 window_floor(z=0)/window_head(z=2700)，但真实窗常是**齐腰窗 / 落地窗带护栏**，wtype=full 未必可信，z=0 落地窗地面点可能无清晰对应物。b3：**标定优先用结构可靠点（墙角 z=0 + 天花板角 z=2700），窗特征降级为辅助/可跳过**；或按 wtype 置信度过滤。**开工前调查**：wtype 在 geometry 数据如何判定（数据准不准）、能否在派生层稳健化而不改 data/projects（红线）。

### D4 对应关系 UX 加固（Prong 1，按需/可分期）
降低"标签↔像素"镜像错误：朝向锁定（先点 1 个无歧义特征定朝向）/ 照片侧目标区域高亮（用粗相机先验）/ 天花板角与其地面孪生联动提示。**开工前调查**：v_i↔yaw 映射（b1 spec §D7 声明未实证）是否可用于朝向先验；不可用则退到"先点无歧义特征"方案。

### D5 3D / 简模引导路线评估（Prong 2，spike）
复用 b1 spike 工具（`scripts/spike/`）+ 真实病例照（只读拉取，含**标定不了的 r_guest2**）：评估在**粗相机 / 无精确标定**下，3D/简模引导能否产出可用实拍图。产出 `docs/test-reports/calib-cure-b3-3dguide-eval-YYYYMMDD.md`：go/no-go + b4 建议 + 成本/依赖（VLM 形体评分、样本、curtain 简模——承接 b1 spike 4 条 GO 条件）。**预算**：若需真实出图，用户授权 relay/fal key（同 b1 spike，本地、只读、不碰生产 data）。

### D6 车道与编排
- 快车道单会话（单实例 local）。
- Prong 1（F002-F004 生产件）串行；Prong 2（F005 spike）隔离研究码，可与 Prong 1 并行。
- verifying：隔离 evaluator；**L2（F006）真实浏览器**——退化照片得明确重拍引导、角落照片能标成、3D 评估结论；需用户配合 + 只读重拉病例照（PIPL）。
- 红线：不写 data/projects、不动 floorplan_core、不改 b2 解算内核、门不放宽、PIPL 照片不入 git、不单推 main。
