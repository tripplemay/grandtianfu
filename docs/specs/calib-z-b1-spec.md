# calib-z-b1 — 标定世界 z 轴符号未约束（家具盒朝地下拉伸）

> 批次类型：Bug 修复（几何正确性）。**本 spec 自包含** —— 立项会话上下文不延续，实现/验收所需的全部实证据均在此。
> 来源：render-fix-b1 隔离 Evaluator 超范围独立发现（HIGH），用户 2026-07-15 裁决单独立批。
> 关联：`backlog.json` → `BL-calibrate-z-axis-sign`；前批 signoff `docs/test-reports/render-fix-b1-signoff-2026-07-15.md`。
> **风险提示（CLAUDE.md）：** 属 `floorplan_core`/AIGC 几何·渲染正确性 —— 渲染错隐蔽，本批应升档编排 + 对抗验证。

## 1. 现象与影响

第7步实拍的**彩盒引导图**里，贴墙件（挂画/窗帘）被画在**地板上**而非墙面带；所有家具盒的**垂直体积朝地下拉伸**。后果：

- **模型明智地无视**地板上的"挂画"盒，把画挂到墙上 → 验收器 `evaluate_geometry_lock` 判「盒区外出现新结构」→ **`auto_check` 持续 false alarm**（生产实证：2026-07-15 08:12/08:14 两张 `ok:false, score 0.85, 新边缘坏块 15~20/111`，各重试 2 次）。
- 家具**形体/高度**引导对 z 反转的照片全部失真（仅 footprint 正确）。
- 旁证：`main._geometry_lock_prompt` 里那句 `"...not a freestanding object on the floor"` 正是在对冲本 bug。

**平面 footprint 不受影响** —— 地面点 z=0，z 列恒被乘 0，故 render-fix-b1 修好的餐桌落位是真的好了（生产两张新图已确认）。**本批只治垂直维度。**

## 2. 根因（已实证部分）

`apps/api/aigc/perspective.py::calibrate()` L146-158：

```python
for sx in (1, -1):
    for sy in (1, -1):
        R = np.column_stack([sx * ex, sy * ey, np.cross(sx * ex, sy * ey)])   # ← z 列是派生的
        t, lams = solve_t(R)
        if np.any(lams <= 0):            # 约束①: 锚点须在相机前方
            continue
        err = 0.0
        for i in range(n):               # 约束②: 锚点重投影误差
            uv = K @ (R @ aw[i] + t); uv = uv[:2] / uv[2]
            err += float(np.hypot(*(uv - ap[i][:2])))
        if best is None or err < best[0]:
            best = (err, R, t)
```

**z 方向从未被任何约束钉过：**

1. z 列 = `cross(sx·ex, sy·ey)` = `sx·sy·cross(ex, ey)` —— 是 x/y 符号的**副产品**，非独立自由度。
2. 两条打分约束**全部只用地面锚点**。**生产 5 份标定的锚点 z 值集合实测全为 `[0]`**（全在地面）。对地面点 `w=(x,y,0)`，`R @ w = sx·ex·x + sy·ey·y` —— **z 列恒被乘 0，对打分零贡献**。
3. ⇒ 最终 z 朝上还是朝下，取决于哪个 `(sx,sy)` 恰好赢下地面拟合，**与物理无关**。

### 2.2 待 Generator 定论的假设（**不得当结论直接实现**）

**`det(R) ≡ 1.000`（5/5 实测）**：因 z 列由前两列叉积派生，构造上恒为右手系（det=+1）。而模块 docstring 的世界约定是 `X=毫米东(+), Y=毫米南(+), Z=上(+)` —— **(East, South, Up) 是左手系**（East×South = Down）。若成立，则 `cross(x_col, y_col)` 给出的是 **Down** 而非 Up，z 列系统性取反。

**但该假设与"2/5 恰好朝上"表面矛盾**（若系统性取反应 5/5 全反）。可能解释：赢下地面拟合的 `(sx,sy)` 因房间/连线分组不同而变，`sx·sy` 随之变号。**此处必须 pre-impl 审计定论**（见 §5），不得凭本节猜测直接改符号 —— 改错方向会让 2/5 正确的标定反而变错。

## 4. 关键设计决策

### D1 — 判别式：相机必须在地板上方（`C_z > 0`）

这是地面锚点**给不了**、而物理**必然成立**的约束：室内拍照者不可能在地板下方。

`C = -Rᵀ t`；要求 `C[2] > 0`。**生产 5/5 完美分离**（3 个负值 = 3 个 z 朝下；2 个正值 = 2 个正确），无需阈值调参。

合理性旁证：2 份正确标定的相机高度 +874mm / +1515mm 均在人手持/三脚架合理区间。

**接入点：** `calibrate()` 的候选筛选（L150 `lams<=0` 旁边），作为**同级硬约束**：`C_z <= 0` 的候选直接 `continue`。若 4 个候选全被排除 → 明确 `raise`（诚实报错优于产出朝地下的相机）。

### D2 — 不得用"事后翻 z 列"的写法

`R' = R @ diag(1,1,-1)` 会让 `det(R') = -1` —— 不再是合法旋转，会污染下游一切依赖 R 正交性的运算。**修必须落在候选选择（或世界系约定/构造式）本身**，不得在解出后打补丁翻符号。

### D3 — 存量标定自愈：重跑而非重标

存储载荷 `photo.calibration` **完整保留了原始输入**（生产实证含 `x_lines` / `y_lines` / `anchors` / `img_wh`，见 §2.1 任一条）。⇒ 修好 `calibrate()` 后**用存量输入重跑即可导出正确 camera，无需用户重新标定**。

**须核实（铁律 1）：** `main._calibration_stale_reason` 的 binding 指纹语义 —— 自愈重写 `camera` 后是否会被判 stale（若会，须同步处理，否则用户看到"标定失效"提示）。

**迁移形态待 Generator 提议、Planner 裁决：** 一次性脚本 / 读取时惰性自愈 / 端点触发。倾向**一次性脚本 + 幂等**（生产只有 5 份，可控且可核对）。**生产写操作须用户明确授权**（CLAUDE.md 生产红线），故脚本先在本地对生产数据**副本**验证，实际执行前停下报用户。

### D4 — byte-safe 边界

本批改 `calibrate()` 输出与存量标定 → **会改变 3 份标定的 camera，进而改变引导图**。这是**预期的行为变更**，非回归。但：

- 2 份本就正确的标定（`dabcb9513905` / `1537e6d83950`）**修后必须逐字节不变** —— 这是"没改错方向"的关键反证。
- `_box_polys` / `annotate_boxes` / `footprint_mask` **不改**（render-fix-b1 刚验收过，本批零触碰）。
- 地面 footprint 投影**修前修后必须一致**（z 列不参与 z=0 平面）—— 这是"只治垂直维度、没动平面落位"的反证。

## 5. 开工前必做（Generator）

**本批 z 符号方向判断错会让 2/5 正确的标定反而变错，且渲染错隐蔽。** 故 Generator 开工前须提交 pre-impl 审计（`framework/harness/pre-impl-adjudication.md` 格式，落 `docs/specs/calib-z-b1-F001-*.md`），至少回答：

1. §2.2 的左手系假设是否成立？`(East, South, Up)` 与 `column_stack([x, y, cross(x,y)])` 的 det=+1 构造是否矛盾？
2. 若矛盾，为何 2/5 仍朝上？（用那 2 份的 `(sx,sy)`/VP 实测复算回答，不得推测）
3. 修法落点：候选筛选加 `C_z>0` 是否足够？会不会出现"4 个候选全不满足"或"满足 `C_z>0` 的候选地面拟合明显更差"？——**用 5 份生产标定的原始输入逐份重跑验证**（`x_lines`/`y_lines`/`anchors` 都在载荷里）。
4. 世界系约定本身是否该改（如 `Y=北`）？若改，影响面（`floorplan_core` 全链路 / golden / 存量数据）多大？—— 倾向**不动约定**，但须给出理由。

Planner 裁决后再开工。

## 6. 验收要点（Evaluator，F003）

1. **5/5 生产标定 `C_z > 0`**（用存量输入重跑 + 自愈后的存量数据双路核对）。
2. **投影目检**：任一地面点投到 `z=+2700` 落**画面上部（天花板方向）**、`z=-2700` 落下部。建议叠回真实空房照目检（前批 Evaluator 用过此法，有效）。
3. **`wall_art` 盒回到墙面带**（v 落画面中上部，非 §2.1 实测的 v≈1009..1118 地面带）。
4. **反证（防止改错方向）**：2 份本就正确的标定（`dabcb9513905` / `1537e6d83950`）修后 camera **逐字节不变**。
5. **反证（只治垂直）**：地面 footprint 投影修前修后一致；render-fix-b1 的 F001/F002/F003 行为未回归（两套 pytest + golden）。
6. **下游 false alarm 是否消除**：用生产实物重跑 `evaluate_geometry_lock`，确认「盒区外出现新结构」不再因挂画上墙而误报。若仍报，如实记录并判断是否属本批范围。
7. **[L2] 真实 AI 出图**：无 key 需授权计费 → 与 decor-b2/render-fix-b1 同降级口径，走确定性几何 + 引导图重生目检；如未执行须如实记 [L2]，**并建议用户部署后人工目检一张实拍图闭环**。
