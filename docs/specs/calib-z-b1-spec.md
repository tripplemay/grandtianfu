第7步实拍的**彩盒引导图**里，贴墙件（挂画/窗帘）被画在**地板上**而非墙面带；所有家具盒的**垂直体积朝地下拉伸**。后果：

## 2. 根因（已实证部分）

`apps/api/aigc/perspective.py::calibrate()` L146-158：

```python
for sx in (1, -1):
    for sy in (1, -1):
        R = np.column_stack([sx * ex, sy * ey, np.cross(sx * ex, sy * ey)])   # ← z 列是派生的
        t, lams = solve_t(R)
        if np.any(lams <= 0):            # 约束①: 锚点须在相机前方
            continue
        err = 0.0
        for i in range(n):               # 约束②: 锚点重投影误差
            uv = K @ (R @ aw[i] + t); uv = uv[:2] / uv[2]
            err += float(np.hypot(*(uv - ap[i][:2])))
        if best is None or err < best[0]:
            best = (err, R, t)
```

**z 方向从未被任何约束钉过：**

1. z 列 = `cross(sx·ex, sy·ey)` = `sx·sy·cross(ex, ey)` —— 是 x/y 符号的**副产品**，非独立自由度。
2. 两条打分约束**全部只用地面锚点**。**生产 11 条标定的锚点 z 值集合实测全为 `[0]`**（全在地面）。对地面点 `w=(x,y,0)`，`R @ w = sx·ex·x + sy·ey·y` —— **z 列恒被乘 0，对打分零贡献**。
3. ⇒ 最终 z 朝上还是朝下，取决于哪个 `(sx,sy)` 恰好赢下地面拟合，**与物理无关**。

### 2.2 左手系根因（**假设已由 F001 审计证明，现为结论**）

> **2026-07-15：** 本节原为"待定论的假设"，已由 `docs/specs/calib-z-b1-F001-z-axis-audit.md` §1 用**合成真值控制实验**证明。以下为结论。

模块 docstring 的世界约定 `X=毫米东(+), Y=毫米南(+), Z=上(+)` —— **(East, South, Up) 是左手系**（East×South = Down）。相机系 `(x=右, y=下, z=前)` 是右手系。⇒ **物理正确的 world→camera `R` 必然 det = −1**。

而 z 列 = `cross(x_col, y_col)` 的构造**强制 det=+1** ⇒ 当 x/y 列拟合正确时，z 列给出的是 **Down** 而非 Up，**系统性取反**。

**合成真值证明**（不依赖生产数据 —— 生产 2 锚点样本本身有歧义）：构造真值相机 `C=(5000,2000,+1500)`、`det(R_true)=−1`，渲染其墙线/锚点喂给真实 `calibrate()` →

```
现状返回  det=+1.000000   相机中心 (5000, 2000, -1500)   <- 地板下方
  col0(East)  误差 1.45e-13  ← 精确恢复
  col1(South) 误差 1.47e-13  ← 精确恢复
  col2(Up)    与真值取反误差 9.55e-14  ← 恰为真值的负数 = 系统性翻转
```

**`det(R)≡1.000`（生产 11/11）不是"正确"的证据 —— 它就是病症本身。**

**"2/5 恰好朝上"的矛盾已解开**（审计 §2）：那不是物理属性，而是 2 锚点下**精确平局的浮点抛硬币**，且 det=+1 下"z 朝上 ⟺ 平面被镜像"。无歧义对照 `bcc615`（3 锚点、err=0.000 精确拟合、唯一候选）恰恰是反转的 —— 约束充分时必然反转。

### 2.3 `dabcb` / `1537e` 究竟被改了什么：水平镜像，不是 z 翻转（Evaluator R3）

> **2026-07-15 补：** 本节是隔离 Evaluator 独立推出的**承重事实**，原 spec 缺失，易让人误判这 4 条的性质。

对这 4 条（`dabcb`×2 / `1537e`×2），修前修后的关系是：

```

```

**不是 z 翻转** —— 这 4 条的**垂直方向本就正确**（相机 DOWN 轴距重力仅 1.07° / 4.79°）。所以：

## 4. 关键设计决策

### D1 — 修构造式（`z = -cross`，det=−1）+ `C_z > 0` 硬门

> **⚠ 2026-07-15 重写（Planner 裁决 #1:A / #2:A）。** 原 D1 = "保持现构造式，只在候选筛选加 `C_z>0`" —— 已被生产实测**证伪且判定有害**（审计 §3）：v6/v7 `bcc615` **4 候选全排除 → raise**（在用 baseline 报废）；v1 `bcc615` 取 err **6634px**（差 34.5 倍）垃圾解；`417ae` 取到**镜像候选**致地面偏 **1058px** → **会把 render-fix-b1 刚修好、用户已在生产确认的餐桌落位再次弄坏**。根因：det=+1 构造下"z 朝上"只能靠镜像换，故 `C_z>0` 在该构造下**恰好是在挑镜像解**。

**修法（两处，缺一不可）：**

```python
# 1) 构造式: Up = -(East x South) —— 尊重 (East,South,Up) 的左手性, det(R) = -1
R = np.column_stack([sx * ex, sy * ey, -np.cross(sx * ex, sy * ey)])
t, lams = solve_t(R)
if np.any(lams <= 0):            # 约束①(既有): 锚点须在相机前方
    continue
# 2) 同级硬约束: 相机必须在地板上方 —— 地面锚点给不了、而物理必然成立
if float((-R.T @ t)[2]) <= 0:
    continue
```

4 个候选全被排除 → 明确 `raise`（诚实报错优于产出朝地下的相机）。

**为何两处缺一不可：** 只改构造式 → 2 锚点的平局仍在，z 方向仍由浮点噪声抛硬币；只加门（原 D1）→ 见上，破坏生产。**合用后：生产 11/11 门后存活候选恰好 1 个 → 完全确定，噪声抛硬币被根除。**

**验证（审计 §3.1）：** 合成真值**精确恢复**（`max|R_fix−R_true| = 1.47e-13`，`C=(5000,2000,+1500)`，det=−1）；生产 11/11 `C_z>0` + 11/11 垂直方向正确 + `R` 精确正交（`max|RᵀR−I| ≤ 3.3e-16`）。

### D2 — 不得用"事后翻 z 列"的写法（**禁令保留，理由已改写**）

**禁令：** 修必须落在**候选选择或构造式本身**，不得在解出后打补丁翻符号（如 `R' = R @ diag(1,1,-1)`）。D1 的写法改的是构造式，**符合本禁令**。

> **⚠ 2026-07-15 理由订正（Planner 裁决 #2:A）：** 原理由称 `det=-1` "不再是合法旋转, 会污染下游一切依赖 R 正交性的运算" —— **两点均不成立**（审计 §5.4）：(a) 负一列**精确保正交**（实测 `max|RᵀR−I| ≤ 3.3e-16`），只改**手性**不改正交性；(b) 下游**无任何代码**依赖 det/手性 —— `cam.R` 的全部用法只有 `R @ w + t`（`perspective.py:32/199/263`），`linalg.inv` 只作用于 `K` 从不作用于 `R`。
>
> **且 `det(R) = -1` 不是缺陷，而是左手系约定的必然要求**（§2.2 已证真值 `det(R_true) = -1`）。
>
> **禁令的真实理由（改写后）：** 事后翻符号是**症状补丁** —— 它把"构造式与世界系约定不自洽"这一根因留在原地，下一个读代码的人会把它"修正"回去。故修在构造式。

存储载荷 `photo.calibration` **完整保留了原始输入**（生产实证含 `x_lines` / `y_lines` / `anchors` / `img_wh`，见 §2.1 任一条）。⇒ 修好 `calibrate()` 后**用存量输入重跑即可导出正确 camera，无需用户重新标定**。
