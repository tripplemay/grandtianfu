## 1. Q1 — 左手系假设是否成立？

**成立，已证明。** 不依赖生产数据（生产 2 锚点样本本身有歧义，见 Q2），而用**已知真值的合成相机**反推。

`(East, South, Up)` 是左手系：`East × South = Down ≠ Up`。相机系 `(x=右, y=下, z=前)` 是右手系。⇒ 物理正确的 world→camera `R` 必然 **det = −1**。

`audit_synthetic_control.py`：构造真值相机 `C=(5000, 2000, +1500)`（地板上方 1.5m），渲染其墙线/锚点，喂给**真实的 `calibrate()`**：

```
真值   det(R_true) = -1.000000          相机中心 (5000, 2000, +1500)
现状   det(R)      = +1.000000          相机中心 (5000, 2000, -1500)   <- 地板下方
  col0 (世界X=East)  误差 1.45e-13   <- 精确恢复
  col1 (世界Y=South) 误差 1.47e-13   <- 精确恢复
  col2 (世界Z=Up)    误差 1.96e+00   <- 唯一错的列
  col2 vs 真值取反   误差 9.55e-14   <- 恰好是真值的负数 = 系统性取反
  地面点 z=0 -> v=884 ; z=+2700 -> v=1588（往下 = 天花板被画在地上）
```

**x/y 拟合是对的，唯独 z 列被系统性取反。** `det(R)≡1.000`（生产 11/11）不是"正确"的证据 —— 它就是病症本身：`z = cross(x,y)` 的构造**强制** det=+1，而正确答案要求 det=−1，故 z 列必然翻转。

## 2. Q2 — 若矛盾，为何 2/5 仍朝上？（spec 要求实测回答，不得推测）

**因为那是浮点噪声的抛硬币结果，不是物理属性 —— 且"z 朝上"是用"平面被镜像"换来的。**

### 2.1 精确平局（实测全精度）

生产 4 份 2 锚点标定，通过 `lams>0` 的候选恰好 2 个，重投影 err **完全平局**：

| photo | 候选 A err | 候选 B err | 相对差 |
|---|---|---|---|
| `dabcb9513905` | 9.864743636744953 | 9.864743636742464 | **2.5e-13** |
| `1537e6d83950` | 195.69854680886166 | 195.6985468088639 | **1.1e-14** |
| `417ae5589afe` | 46.37522839708953 | 46.37522839708952 | **1.5e-16** |
| `ae8e5b875fd9` | 178.13224749335797 | 178.13224749335217 | **3.3e-14** |

两个候选的 `C_z` 恰好**符号相反**。胜负由 `err < best[0]` 在 **1e-13 级噪声**上裁定 ⇒ z 朝上朝下是抛硬币。

### 2.2 跨机器不可复现 —— 平局的铁证

**`1537e6d83950` 的生产存量 camera 与本机重算结果不一致：`max|ΔR| = 2.0`**（其余 10 份 ΔR ≤ 8e-13）。**同一份存储输入，换台机器（BLAS 实现不同）就得到相反的 z。** spec §2.1 表里记录的"2/5 正确"只是生产那次的抽签结果。

### 2.3 平局的机理：z 朝上 ⟺ 平面被镜像（二者是同一事件）

翻 `sy` = 把世界 +Y 从"南"改判成"北"。2 个锚点无法区分（平局已证）。而 det=+1 构造下 `z = cross(x,y)`：

- 若 cols(0,1) = (East, South) 正确 → `z = cross(E,S) = Down` → **z 必朝下**
- 若 cols(0,1) = (East, North) 镜像 → `z = cross(E,N) = Up` → **z 朝上，但平面在 Y 上镜像**

（(W,N) 是 180° 旋转 → `cross(W,N)=cross(E,S)=Down` 仍朝下。故 **det=+1 且 z 朝上 ⟺ 恰好一个轴被镜像**。）

⇒ **spec 认定"本就正确"的那 2 份（`dabcb` / `1537e`），是用镜像的平面图换来的朝上 z。**

### 2.4 生产里的无歧义对照：`bcc615`（3 锚点）

`bcc615` 有 **3 个锚点** → 无平局（两候选 err **192.2 vs 6634.5**，量级差异）→ 唯一候选、**err = 0.000（精确拟合）** → 而它 `C_z = −2266.8`。

**约束充分、拟合完美的那一份，恰恰是反转的。** 这在真实数据上钉死了 §1 的理论。

## 4. Q4 — 世界系约定本身是否该改（如 Y=北）？

**不改。** 理由：

1. **约定不是 bug，构造式才是。** 文档约定 `(X=东, Y=南, Z=上)` 与 `floorplan_core` 的 px→mm 映射自洽（图像 y 向下 = 向南）。矛盾的是 `cross(x,y)` 强制 det=+1，与该约定的左手性冲突。改构造式一行即可自洽。
2. **改约定的影响面不可接受：** 会镜像**每一份**存量几何/家具/baseline，作废全部 golden 与生产数据，且 web 端坐标语义全变 —— 为零收益付出全链路代价。

## 5. 超出 4 问的范围订正（须一并裁决）

### 5.1 样本是 11 份，不是 5 份 —— spec 漏了整个 v7

生产实际含标定：**v1 ×1 + v6 ×5 + v7 ×5 = 11 条**。spec §2.1 只列 5 条，**完全漏掉 v7 —— 而 v7 正是在用的 baseline**（render-fix-b1 的"户型 v7/胡桃石韵轻奢"）。

另：v6/v7 的 `bcc615` 与 v1 的 `bcc615` 是**不同的标定**（C_z −2266.8 vs −2427.1；err 0.000 vs 192.2）。spec 把 `bcc615` 只归给 v1 会漏改。

### 5.2 F002 「存量 3 份反转」口径错误

按存量 `C_z` 实际分类：

- **z 朝下 7 条**：v1/`bcc615`、v6/`bcc615`、v6/`417ae`、v6/`ae8e5`、v7/`bcc615`、v7/`417ae`、v7/`ae8e5`
- **z 朝上但平面镜像 4 条**：v6/`dabcb`、v7/`dabcb`、v6/`1537e`、v7/`1537e`

**11 条全部需要重跑**，其中 4 条（`dabcb`/`1537e` × 2 baseline）**平面会移动 ~1.5–1.7k px** —— 这是主卧 + 衣帽间的用户可见变化。

### 5.3 D3 的 stale 核实 —— 已完成，结论：不会判 stale

`main.py:894 _calibration_stale_reason` 的 binding 指纹只含 `room_id` + `room_rect_hash`，**camera 不在指纹内** ⇒ 自愈重写 `camera` **不会**被判 stale，用户不会看到"标定失效"。无需额外处理。

### 5.4 D2 的理由必须改写（结论不变，理由是错的）

D2 称 `det=-1` "不再是合法旋转, 会污染下游一切依赖 R 正交性的运算" —— **两点都不成立**：

1. **负一列精确保正交**：实测 `max|RᵀR − I| ≤ 3.3e-16`。负列只改**手性**，不改正交性。
2. **下游无任何代码依赖 det/手性**：`cam.R` 的全部用法只有 `R @ w + t`（`perspective.py:32 / 199 / 263`）；`linalg.inv` 只作用于 `K`，从不作用于 `R`。

且 `det=-1` 不是缺陷 —— 它是左手系约定**要求**的（§1 已证 `det(R_true) = -1`）。**D2 的禁令（不得事后打补丁翻符号）保留，理由须改写。**

## 6. 诚实边界 / 残余不确定性

1. **`dabcb` / `1537e` 的平面移动缺少独立视觉确认。** 2 锚点下存量数据**在数学上无法**区分两个候选（对锚点重投影完全等价）。修法靠物理约束定夺，合成真值已证其正确；但我尝试的两个独立代理判据**互相矛盾**（"房间地面四角在画幅内"偏向 stored/dabcb、fixed/1537e；"相机中心在房内"偏向 fixed/dabcb、stored/1537e），且**后者被对照样本证伪**（`bcc615` n=3、err=0.000、无歧义，其相机也在房外 —— 隔门取景本就正常）。⇒ **代理判据不可用，须 Evaluator 用真实照片目检确认（F003）。**
2. **根治歧义需 ≥3 个不共线锚点**（`bcc615` 实证 n=3 决定性破平局：192 vs 6634）。当前 4 份 n=2 标定的 (sx,sy) 本质上是靠物理约束而非数据定的。是否强制 ≥3 锚点，见决议 #5。
3. **`ae8e5`（r_garden）标定质量本就差**：修后 `C_z = +156mm`（相机离地 15.6cm，不合常理），锚点 err 178px。与本 bug 无关，建议登记 soft-watch 而非阻断。
