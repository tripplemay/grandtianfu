# 阅天府软装工具 · 前端 UI/UX 审查 + 工作台框架(Workbench Shell)实施计划

> 本工作流仅做审查 / 设计 / 规划,**不写实现代码**。以下为可落地的最终实施计划。

---

## ① 需求重述

**A. UI/UX 审查范围**
对 `/studio` 三个现有页面在 desktop(1440)/mobile(390)下做全面体验审查:
- 项目台 `/studio/projects`
- 编辑器 `/studio/projects/[id]/editor`(几何/家具双模)
- 画廊 `/studio/projects/[id]/gallery`(三引擎视图)
覆盖维度:应用外壳缺失、信息架构与导航、页面布局/标题规范、品牌一致性、暗色/主题、响应式、空/载/错态、反馈(toast/确认)、可访问性、跨页流程断点。

**B. 工作台框架要达成什么**
为 `/studio` 建立一套**自有的两级应用外壳**(全局壳 + 项目内壳),复用 Horizon 的 Sidebar/Navbar/Footer/主题机制,但**换数据源、换品牌、去 demo 痕迹**;不让 studio 去蹭 `/admin` 那套写死演示内容的壳。最终交付:
- 全局侧栏(Sidebar)+ 顶栏(Navbar)+ 页脚 + 主题/暗色切换 + 用户区占位
- 统一页面模板(消除"标题贴顶"与三页各写容器)
- 统一导航 IA(全局侧栏 + 项目内分区 + 面包屑)
- 统一状态/反馈规范(空/载/错 + toast + 确认弹窗)

**C. 为哪些未来功能铺底座**
- #4 软装风格对话/方案(项目作用域)
- #6 AI 写实效果图(项目作用域)
- 设置(全局作用域)
- 后续:多户型工作区切换、账户(预留 DOM 位与导航槽,不实现逻辑)
核心价值 = **"零摩擦接入"**:未来加一个功能 = 注册一条路由 + 新建一个套 `<PageShell>` 的页面,外壳/侧栏/面包屑/主题/响应式/留白自动获得。

**关键现状(已逐文件核实)**
- `src/app/studio/` 下**没有 `layout.tsx`**,三页裸渲染,各自 `mx-auto max-w-[1400px] px-4 py-6` + `<h1>` 直接贴视口顶。
- `src/app/layout.tsx` 把 `<body className="dark">` 写死,`<html lang="en">`(界面是中文)。
- Horizon 完整外壳只挂 `/admin`(`src/app/admin/layout.tsx`);`src/routes.tsx` 全是 NFT/Dashboards demo,无一条 studio。
- 品牌写死:"Horizon PRO"、"Adela Parkson"、"Buy Horizon UI PRO"。
- 反馈不统一:编辑器有自研 `useToast`+`StatusLines`+`BackendErrorBanner`+`LoadStateBadge`;项目台删除用原生 `window.confirm`。
- 移动端横向溢出、画布裁切、文案截断;渲染图无 onError/骨架兜底。

## ② 工作台框架设计定稿

### 2.1 应用外壳方案(复用 Horizon)

**复用策略:复制 `admin/layout.tsx` 骨架 → 新建 `studio/layout.tsx`,组件层新建 studio 专用薄封装(`components/studio/shell/*`),复用 `Links`/`Card`/`Scrollbar`/`Dropdown`/`Portal` 等原子。不改 `/admin` demo,向后兼容。**

骨架(沿用 admin 已验证的组装):
```
<div flex h-full w-full>
  <StudioSidebar routes={studioRoutes} open hovered mini brand={阅天府} />   // fixed, z-50, xl 起常驻 / xl 下抽屉
  <div h-full w-full>
    <main className="... xl:ml-[313px] (mini 时 xl:ml-[142px])">
      <Portal><StudioNavbar breadcrumb={…} theme setTheme onOpenSidenav … /></Portal>
      <div className="!pt-[100px]">{children}</div>     // 给 fixed Navbar 让位 → 彻底解决"标题贴顶"
      <StudioFooter/>
    </main>
  </div>
</div>
```
- 复用 `ConfiguratorContext`(已在根 `AppWrappers` 全局提供 mini/hovered/theme):studio 自动可消费,**不新增 ChakraProvider / Provider**。
- 响应式抽屉、mini/hover 缩进逻辑直接照搬 admin。
- `'use client'`(用 `usePathname`/`useContext`/`document`)。

### 2.2 导航 IA(全局侧栏 + 项目内分区 + 面包屑)

**两级 + 两作用域**:
- **全局作用域 `/studio/*`(全局壳侧栏顶层)**:精简
  - 项目台 `/studio/projects`
  - 设置 `/studio/settings`(占位,`comingSoon`)
  - (预留)账户 / 多户型工作区
- **项目作用域 `/studio/projects/[id]/*`(侧栏动态"当前项目"分组)**:命中 `[id]` 时插入
  - 编辑器 `/editor`、画廊 `/gallery`
  - 软装方案 `/scheme`(#4,占位)、效果图 `/render`(#6,占位)

**IA 判定**:#4/#6 都针对某个户型 → 归项目作用域(侧栏垂直列表天然可扩展、不溢出),**不放全局顶层**。

**ProjectTabs 处置**:废弃(职责被"侧栏当前项目分组 + 面包屑"接管),避免水平 tab 在 #4/#6 加入后溢出 + 双导航困惑。可分两阶段:P2 先保留为内容区快捷切换,P2 末移除。

### 2.3 路由结构(目标态)
```
src/app/studio/
  layout.tsx              ← 新建:全局壳
  page.tsx               ← 新建:redirect → /studio/projects
  projects/page.tsx       ← 改:套 PageShell
  projects/[id]/
    layout.tsx           ← 改:去 ProjectTabs,注入 ProjectNavContext(id+名),保留 generateStaticParams
    editor/page.tsx       ← 改:套 PageShell(专注/满高变体)
    gallery/page.tsx      ← 改:套 PageShell
    scheme/page.tsx       ← 未来 #4(占位)
    render/page.tsx       ← 未来 #6(占位)
  settings/page.tsx       ← 未来(占位)
```
新增支撑:`src/lib/studioRoutes.tsx`(IRoute[],与 demo `routes.tsx` 隔离)、`src/components/studio/shell/{StudioSidebar,StudioNavbar,StudioFooter,StudioBreadcrumb,ProjectNavContext}.tsx`、`src/components/studio/ui/{PageShell,EmptyState,LoadingState,ConfirmDialog,ToastHost}.tsx`。

**层级**:`studio/layout.tsx`(壳渲染一次)→ `[id]/layout.tsx`(只注入上下文/面包屑参数,**不重复渲染壳**)→ `page.tsx`(`<PageShell>` 包内容)。

### 2.4 页面布局规范(PageShell 模板)
统一所有内容页,容器单一来源:
```
<PageShell title description actions aside state>{children}</PageShell>
```
- 容器:`mx-auto w-full max-w-[1400px] px-4 py-6`(从三页抽出)。
- 标题区:`h1 text-2xl font-bold text-navy-700 dark:text-white` + 副标题 `text-sm text-gray-600 dark:text-gray-300`。
- 工具区:标题行右侧 flex(收编"＋新建项目"、保存等)。
- 内容/侧栏:`flex flex-col gap-4 lg:flex-row`,aside 用现有 `SidePanel`(max-w 340)。
- editor 用"满高内容区"变体(`h-[calc(100vh-…)]`),并可借 `mini` 折叠侧栏做"专注模式"。

### 2.5 品牌
- 侧栏 "Horizon PRO" → "阅天府软装"(mini 态 "阅"/logo);通过给 Sidebar 加 `brand?:{full,mini}` prop 实现,不 fork。
- 删 `SidebarCard`(Buy Horizon PRO 理财卡);用户区 "Adela/Product Designer" → 静态"工作台用户"占位 + 退出位(不接逻辑)。
- Navbar 删 Notification/Info demo 下拉、"Buy Horizon UI PRO";Footer "© Horizon UI/simmmple" → 阅天府版权。

### 2.6 主题 / 响应式 / 空载错态规范
- **暗色**:收敛为**单一来源**(Navbar 开关读写 `body.dark` + localStorage 持久化);Root `layout.tsx` 去掉写死 `dark`,改 SSR 安全初始化(`suppressHydrationWarning` + 首屏 inline script 读 localStorage,规避闪烁)。studio 组件已普遍写 `dark:` 变体。
- **响应式**:外壳 `xl` 起常驻 + `xl:ml-[313px]`,`xl` 下抽屉 + 汉堡键;内容 `flex-col lg:flex-row`;卡片网格 `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4`;画布 `max-w-full overflow-x`。
- **空/载/错**:`EmptyState`(icon/title/desc/action)、`LoadingState`(骨架 + spinner 两形态)、页面级失败用 `BackendErrorBanner`、字段/校验级用 `StatusLines`(保留 `LoadStateBadge`)。
- **toast/确认**:`useToast` 提升为壳级 `ToastHost`(放 `studio/layout.tsx`),`useToastContext()` 全局可用;`ConfirmDialog`(`confirm()→Promise<boolean>`)替换 `window.confirm`。

## ④ 分阶段实施步骤

> 原则:**先搭外壳框架 → 再统一页面规范 → 再补状态与打磨**。每阶段结束 `yarn build`(含 `output:export`)绿 + 无头 Chrome 截图回归(desktop+mobile),**不破坏现编辑器/引擎/数据**。

**Phase 1 — 外壳骨架(复杂度 High)**
范围:`studio/layout.tsx`(Sidebar+Navbar+Footer+`!pt-[100px]`)、`studio/page.tsx` redirect、`studioRoutes.tsx`(仅"项目台")、`StudioSidebar/StudioNavbar/StudioFooter` 薄封装(换品牌、删 demo、加 `brand` prop)、扩展 `Links.tsx` 接受 `/studio` layout 分支、根 `page.tsx` redirect 改 `/studio/projects`。
产出:三现有页进入即有侧栏/顶栏/品牌/页脚,标题不再贴顶。
验收:`yarn build` 绿;三页 desktop/mobile 截图均有外壳;`/admin` demo 无回归;编辑器/画廊功能不变。

**Phase 2 — 页面模板 + 项目内导航(复杂度 Med)**
范围:`PageShell` 套三页(editor 满高变体);`ProjectNavContext`(在 `[id]/layout.tsx` 注入 id+名,保留 `generateStaticParams`)+ 侧栏"当前项目"动态分组 + `StudioBreadcrumb`;废弃/降级 `ProjectTabs`;消除双容器/双 padding。
产出:统一头部规范、面包屑、侧栏项目分区;`#4/#6` 槽位以 `comingSoon` 占位可见。
验收:三页头部一致、面包屑正确、侧栏当前项目高亮;移动端无溢出;build 绿 + 截图回归。

**Phase 3 — 状态 / 反馈规范(复杂度 Med)**
范围:`EmptyState/LoadingState/ConfirmDialog`、壳级 `ToastHost`+`useToastContext`、`<RenderImage>`(骨架+onError);替换 `window.confirm`,统一 toast,三处渲染图接错误兜底。
产出:全 studio 空/载/错/确认/通知统一。
验收:断开 api(模拟)时图片优雅降级、删除走 ConfirmDialog、操作有 toast;build 绿 + 截图。

**Phase 4 — 主题收敛 + a11y(复杂度 Med)**
范围:暗色单一来源 + Root 去写死 + localStorage 持久化 + inline script 防闪烁;`lang="zh-CN"`;可点元素 aria;暗色对比校验。
产出:可切明/暗且持久,首屏无闪烁,a11y 基线达标。
验收:刷新保持主题、明暗均可读;build 绿(SSR 不报错)+ 截图(明/暗各一)。

**Phase 5 — 未来位预留验证(复杂度 Low)**
范围:`scheme/render/settings` 占位页(套 PageShell + EmptyState + `comingSoon`);按"零摩擦接入配方"端到端跑通一遍。
产出:导航即点即达占位页,验证加新功能仅需"注册路由 + 套 PageShell"。
验收:点侧栏占位项可达;build 绿 + 截图;确认未触碰外壳/面包屑/主题即获得一致体验。

## ⑤ 依赖、风险、复杂度

**依赖**
- 运行:`apps/api`(8000,GEOM_READONLY)+`next dev`(3000);无头 Chrome 截图。
- 组件:`@chakra-ui/system@2.5.7`(Accordion/Dropdown/Portal/Drawer/useDisclosure)、`ConfiguratorContext`、`utils/navigation.ts`(`getActiveRoute`)、现有 `components/studio/ui/*`、`lib/studioApi.ts`(Envelope/unwrap)。

**风险**
1. **admin 外壳 Chakra/React18 适配**(Med):当前**无顶层 ChakraProvider**,Chakra 组件靠默认主题工作。**严禁新增第二个 ChakraProvider/Context Provider**(双主题/双 darkmode 状态)。照搬 admin 已验证组装即可。
2. **`output:export` 与 layout 嵌套**(Med):`[id]/layout.tsx` 是 server 组件且有 `generateStaticParams`;新 `studio/layout.tsx` 设为 client。Next 允许 client layout 包 server layout,**但必须回归验证 `yarn build`(export)不报错**。
3. **编辑器全宽布局与侧栏冲突**(Med-High):`FloorplanEditor` SVG 需大视口,外壳 `!pt-[100px]`+侧栏缩进压缩可用面积。用 editor 满高变体 + `mini` 专注模式缓和;StageSvg 缩放需在小屏验证。
4. **暗色去写死首屏闪烁**(Low-Med):inline script 提前置 class + `suppressHydrationWarning` 规避。
5. **激活态子串匹配串台**(Low):`/projects` 误命中 `/projects/x`;改前缀分段匹配。
6. **不污染 `/admin` demo**(Low):仅新增 studioRoutes + Studio* 封装 + 可选 `brand`/`comingSoon` prop,向后兼容。

**复杂度汇总**:Phase 1 = High;Phase 2/3/4 = Med;Phase 5 = Low。整体 Med-High,约 2-3 人天(P1≈1,P2≈0.5,P3≈0.5,P4≈0.5,P5≈0.5)。

## ⑥ 不必做 / 暂不做(防过度工程)

- **鉴权 / 多租户 / 账户后端**:仅留用户区 DOM 占位 + 退出位,不接登录/会话逻辑。
- **RTL**:工作台只用 LTR,忽略 `/rtl` 整套。
- **Configurator 主题色调色板**:可暂时移除或折叠,不作为交付重点(只保留暗色切换)。
- **`/admin` demo 重构/清理**:保持原状,不动(降低回归面)。
- **#4 软装对话 / #6 AI 写实的真实功能**:本工作流只建占位页 + 导航槽 + 接入配方,不实现业务/接 AI gateway。
- **后端 `GET /api/projects/{id}` 单项接口**:项目名先用 id/列表兜底,不为面包屑新增后端接口(可选后续)。
- **全站设计系统重做 / 替换 Chakra→其它**:复用现有体系,不引入新 UI 框架。
- **E2E 自动化测试体系**:本阶段以 build 绿 + 截图回归为验收,不搭 Playwright 全套。
