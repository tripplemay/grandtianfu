# image2 回归评测集 (P2)

- 日期: 2026-07-08
- 来源: `docs/AIGC链路与提示词诊断-20260708.md` 第 3 条建议"建立 image2 回归评测集"
- 目的: 每次调 prompt / 链路后, 用**同一批固定输入**横向比较 image2 出图质量, 避免"凭感觉调"。

## 为什么要固定评测集

image2 的确定性测试 (mock) 只能证工程链路, 不能证画质。画质变化只有对着**同一组输入**
反复出图才比得出来。诊断报告把这条列为 P2 —— 不是每次都跑 (真实出图烧额度), 而是在改
提示词/链路后做一次横向回归。

## 评测集构成 (最小集)

声明在 `apps/api/aigc/eval_harness.py::EVAL_CASES`, 共 10 个用例:

| 用例 | 类型 | 说明 |
|------|------|------|
| axon_default | 第5步轴测 | 整宅轴测 · 默认方案 |
| axon_ai_style | 第5步轴测 | 整宅轴测 · AI 风格方案 |
| real_live_v0..v3 | 第7步实拍 | 客餐厅实拍 · 四个拍摄方向各一 |
| real_bed_v0..v1 | 第7步实拍 | 主卧实拍 · 两个方向 |
| wall_ctx_a / b | 第7步实拍 | 方案墙面贴实拍参考图后复跑 (材质C 上下文) |

## 每次记录什么

`metrics_row()` 从每张 render record 自动抽取 (无需人判):

- provider 是否成功出图
- **请求尺寸 vs 实际返回尺寸** 是否一致 (批3 的 `requested_size`/`actual_size`;
  报告实测请求 `1536x1024` 返回 `1677x938` 即 `mismatch`)
- prompt 字数
- token usage
- mode / 风格快照 `style_snapshot`

以下需要评审人回填 (`HUMAN_SCORE_KEYS` / `UNACCEPTABLE_KEYS`, 0-5 分):

- 家具落位评分 `placement_score`
- 结构保持评分 `structure_score`
- 风格符合度评分 `style_score`
- 不可接受错误 (任一命中即判废): 改墙 / 改窗 / 错房间 / 家具漂浮 / 出现人物或文字 / 水印

## 怎么跑

纯指标逻辑可单测 (不联网):

```bash
PYTHONPATH=packages/floorplan_core:apps/api python3 -m pytest apps/api/tests/test_eval_harness.py -q
```

真实出图 (手动, 会烧额度) 用运行器 `apps/api/scripts/run_image2_eval.py`:

1. 起服务 (已配 `OPENAI_API_KEY` / `OPENAI_BASE_URL`), 上传并标注好评测用空房照。
2. 写 `config.json`, 把用例映射到该环境的具体 `scheme_id` / `photo_id`:

   ```json
   {
     "axon_default":  {"scheme_id": "default"},
     "axon_ai_style": {"scheme_id": "scheme_ai_20260708_..."},
     "real_live_v0":  {"scheme_id": "scheme_ai_20260708_...", "photo_id": "<空房照id>"}
   }
   ```

   只需填要跑的用例; 未填的记为 skipped。

3. 运行:

   ```bash
   EVAL_API_BASE=http://127.0.0.1:8000 \
     python3 apps/api/scripts/run_image2_eval.py \
       --project D --config config.json --out artifacts/image2-eval-手动跑
   ```

输出 `summary.json` (自动指标全量) 与 `report.md` (可读表 + 汇总)。评审人在 `report.md`
基础上回填落位/结构/风格评分与不可接受错误, 得到该批完整结论, 与上一批横向对比。

## 与前几批的衔接

- 批3 让 record 有了 `requested_size`/`actual_size` —— 评测集据此自动判尺寸错配。
- 批5 让 record 有了 `style_snapshot` —— 每张出图定格了用的风格, 横向比较不同风格才有依据。
- 后续若把提示词升级为结构化 Prompt Compiler (报告建议 1), 评测集是它的回归护栏。
